Recent migration from single JSON file to JSONL (JSON Lines) format for storing processed transcripts and memories.

**Why JSONL**: Supports streaming processing of large files without loading entire file into memory. Each line is a valid JSON object, allowing line-by-line reading.

**Key classes**:
- `EpisodicJsonlStore` - Stores episodic memories in JSONL format with file padding (6-digit indexes)
- `ThinkingJsonlStore` - Stores thinking memories in JSONL format
- `ProcessedLog` - Tracks which transcripts have been processed (JSONL format)

**File naming**: Uses 6-digit padding for JSONL files (e.g., `episodic-000001.jsonl`, `thinking-000001.jsonl`) to support multiple files if needed.

**Important**: When reading JSONL files, parse line-by-line to handle large files efficiently. Each line must be a complete valid JSON object.
