# Transcript Collector Overview

The transcript-collector is responsible for synchronizing transcripts from Claude's internal cache (`~/.claude/projects/`) into the local-recall system for memory extraction.

## How It Works

1. **Path Resolution**: Converts the user's project `cwd` to the corresponding Claude projects folder
2. **File Discovery**: Scans the Claude projects folder for new or updated transcript files
3. **Change Detection**: Uses content hashing to identify modified transcripts
4. **Sync Operation**: Copies or updates transcript files in the local-recall system
5. **Cleanup**: Removes memories for transcripts that have been deleted

## Integration with MCP Server

The MCP server daemon (running every 5 minutes) uses the transcript-collector to:
1. Get the latest transcripts from Claude's cache
2. Detect which transcripts are new or changed
3. Pass changed transcripts to the memory-extractor for analysis
4. Update or delete memories based on transcript changes

## Key Functions

- `syncTranscripts()` - Main sync operation
- `getTranscriptPath()` - Convert project cwd to Claude's transcript location
- `hashTranscript()` - Create content hash for change detection
- `trackProcessedTranscripts()` - Maintain state of what's been processed
