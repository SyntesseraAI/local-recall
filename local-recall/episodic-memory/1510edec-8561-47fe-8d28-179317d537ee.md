# Transcript JSONL Format

Each `.jsonl` file in `~/.claude/projects/{project}/transcripts/` contains newline-delimited JSON events.

## Event Structure

Each line is a JSON object with these common properties:
- `id` - Unique event ID (UUID or sequential)
- `type` - Event type (e.g., `message`, `tool-call`, `tool-result`)
- `timestamp` - ISO-8601 timestamp
- Additional fields depend on event type

## Event Types

- **message**: User or assistant message
  - `role` - `user` or `assistant`
  - `content` - Message text

- **tool-call**: Tool invocation
  - `toolName` - Name of the tool
  - `input` - Tool parameters (object)
  - `clientToolCallId` - Reference ID

- **tool-result**: Tool output
  - `clientToolCallId` - References the tool-call
  - `content` - Tool output/result
  - `isError` - Boolean indicating if result is an error

## Parsing Considerations

- Files are newline-delimited (one JSON object per line)
- Need to read line-by-line for streaming or split by `\n`
- Timestamps help with deduplication and ordering
- Tool calls and results are linked via `clientToolCallId`
