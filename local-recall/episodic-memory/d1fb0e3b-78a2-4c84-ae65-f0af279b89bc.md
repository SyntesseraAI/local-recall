# Claude Code Transcript Event Structure

## Event Types

Transcripts contain different event types representing interactions:
- `type: "user-message"` - User input
- `type: "assistant-message"` - Claude's response
- `type: "tool-use"` - Tool invocations
- `type: "tool-result"` - Tool execution results
- Other metadata events for session management

## Event Fields

Each event includes:
- `timestamp` - ISO-8601 timestamp of when the event occurred
- `type` - The event type (user-message, assistant-message, tool-use, tool-result)
- `content` - The actual message or tool information
- `session_id` - Session identifier
- `tool_id` (for tool events) - Identifier for the tool execution

## Usage Pattern

For memory extraction:
1. Read `.jsonl` file line-by-line
2. Parse each line as separate JSON object
3. Filter by event type (typically interested in user-message, assistant-message, tool-use, tool-result)
4. Extract relevant information for memory creation
5. Use timestamps for deduplication and sorting
