Local Recall enables persistent memory across Claude Code sessions through structured markdown files. It provides:

1. **Core Components**:
   - Memory Manager (CRUD operations for memory files)
   - Vector Store (SQLite + embeddings for semantic search)
   - Search implementation using vector embeddings
   - Claude Code hooks (SessionStart, UserPromptSubmit, Stop)
   - MCP server for memory tools

2. **Memory File Format**: Each memory is a markdown file with YAML frontmatter containing:
   - `id`: Unique UUID
   - `subject`: Brief description
   - `keywords`: Searchable keywords
   - `applies_to`: Scope (global, file:<path>, area:<name>)
   - `occurred_at`: ISO-8601 timestamp
   - `content_hash`: SHA-256 prefix for deduplication

3. **Key Features**:
   - Version-controlled memory files (tracked in git)
   - SQLite vector database (gitignored)
   - BGE-small-en-v1.5 embedding model for semantic search
   - Deduplication by content_hash and occurred_at
   - Background MCP daemon that syncs transcripts every 5 minutes
