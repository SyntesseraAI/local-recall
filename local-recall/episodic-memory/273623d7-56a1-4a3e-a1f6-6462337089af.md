# User Communication Preferences

## Key Preferences

1. **Be Direct**: User wants straightforward answers without lengthy explanations or speculation
2. **Ask First**: When uncertain about requirements, ask the user directly rather than making assumptions
3. **Avoid Verbose Responses**: User finds lengthy speculative responses unnecessary
4. **Follow Specifications**: User provides clear specifications and expects them to be followed

## Application

When working with this user:
- Keep responses focused and concise
- Ask clarifying questions early if requirements are ambiguous
- Don't speculate about what they "might want"
- Avoid suggesting features beyond what was requested
- Follow provided specifications exactly

## Context

This preference became clear during transcript analysis when the user indicated they prefer getting direct answers to questions rather than lengthy speculative responses about what might be needed.
