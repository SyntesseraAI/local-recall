Each memory in local-recall follows a consistent structure:

- **subject**: Brief one-line description (max 200 characters) that summarizes the memory
- **keywords**: Array of 1-10 searchable lowercase keywords for fuzzy matching
- **applies_to**: Scope indicator - either `global` for codebase-wide knowledge, `file:<path>` for file-specific information, or `area:<name>` for component-related memories
- **content**: Detailed markdown-formatted explanation providing context, reasoning, and specific details

This structure enables efficient searching and helps future sessions quickly find relevant memories through keyword matching.
