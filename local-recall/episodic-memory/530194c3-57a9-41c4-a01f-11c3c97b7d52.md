Memory files use a standardized markdown format with YAML frontmatter for metadata.

## Frontmatter Fields

- `id`: Unique identifier (UUID)
- `subject`: Brief one-line description of the memory
- `keywords`: Array of searchable keywords (crucial for discovery)
- `applies_to`: Scope indicator - `global`, `file:<path>`, or `area:<name>`
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 hash prefix (16 chars) for duplicate detection

## File Location

Memory files are stored in `local-recall/episodic-memory/` directory and are version-controlled in git.

## Important Notes

- Memory files ARE tracked in git (version-controlled)
- The index (`local-recall/index.json`) is gitignored
- Memories are idempotent - duplicates are detected via content_hash and occurred_at
- Each memory should be atomic, focusing on one concept
