Each memory is a markdown file with YAML frontmatter containing:
- `id`: UUID for unique identification
- `subject`: One-line brief description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 prefix (16 chars) for deduplication

Memories follow idempotent pattern - duplicate detection prevents duplicates based on occurred_at and content_hash.
