The memory-extractor.ts module provides core functionality for extracting memories from Claude Code session transcripts. Key responsibilities include:

1. **Transcript Condensing**: Condenses full transcripts into simplified event format with [User], [Assistant], [Tool: Name], and [Result: OK/ERROR] entries
2. **Memory Extraction**: Uses Claude Haiku to analyze condensed transcripts and extract valuable memories
3. **Duplicate Prevention**: Checks for existing memories using content hash and occurred_at timestamp
4. **Error Handling**: Gracefully handles transcript processing failures and logs issues

The module integrates with memory creation pipeline to ensure extracted memories are properly stored with metadata.
