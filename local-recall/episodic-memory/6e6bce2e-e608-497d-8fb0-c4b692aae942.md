# User Import Style Preferences

The user prefers:
- **Relative imports with explicit paths** (e.g., `import { foo } from '../utils/helper'`)
- **Avoid barrel imports** (avoid using index.ts re-exports)
- **Clear, direct import statements** rather than using path aliases or index.ts files

This preference should be applied when generating code that imports from other modules in the project.
