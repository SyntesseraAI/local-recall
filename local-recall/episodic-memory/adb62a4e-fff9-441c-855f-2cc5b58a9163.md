In JSONL transcripts, message content can be either a string or an array of content blocks. Each block has a `type` field (e.g., 'text', 'tool_use') and type-specific fields. Tool use blocks have `id`, `name`, and `input` fields. This flexibility requires robust parsing in the transcript collector.
