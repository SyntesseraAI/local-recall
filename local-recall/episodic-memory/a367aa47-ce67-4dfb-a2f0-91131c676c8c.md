Added field name normalization in the parseClaudeResponse function to handle cases where Claude Haiku returns memory objects with alternative field names (e.g., 'name' instead of 'subject', 'tags' instead of 'keywords'). The normalization maps these alternative names to the standard schema before Zod validation, making the parser more robust to Claude's varying response formats.
