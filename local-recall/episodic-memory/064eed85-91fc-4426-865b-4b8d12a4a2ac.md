# Transcript Extraction Optimization

When extracting memories from transcripts, use keyword filtering approach instead of analyzing the full prompt:

1. **Current Issue**: Passing full transcript/conversation content to Claude Haiku is inefficient and produces low-quality memories

2. **Better Approach**: 
   - Extract key concepts/keywords from the transcript first
   - Filter to only the most relevant keywords using fuzzy matching against existing memory index
   - Use only the filtered keywords (not full content) to guide memory creation

3. **Benefits**:
   - Reduces token usage and API calls
   - Produces more focused, higher-quality memories
   - Prevents duplicate memory creation
   - Speeds up memory extraction process

4. **Implementation**: In `transcript-collector.ts`, modify the transcript processing to filter keywords before memory creation rather than passing raw transcript content.
