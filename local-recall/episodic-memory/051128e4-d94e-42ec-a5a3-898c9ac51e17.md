# Transcript Condensing for Memory Extraction

The `transcript-collector.ts` module implements transcript condensing to improve memory extraction efficiency.

## Purpose

- Reduces verbose transcript output before passing to Claude for memory extraction
- Uses Claude Haiku to intelligently condense transcripts while preserving key information
- Improves token efficiency and focuses memory extraction on relevant events

## Implementation Details

- Condenses transcripts into compact event format:
  - `[User]` - User requests/messages
  - `[Assistant]` - Claude responses/explanations
  - `[Tool: Name]` - Tool invocations with names
  - `[Result: OK/ERROR]` - Outcomes of tool calls

## Integration

- Called before memory extraction to reduce context size
- Helps Claude Haiku generate more focused and relevant memories
- Reduces processing time and API costs
