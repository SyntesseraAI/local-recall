Local Recall maintains two separate vector indexes:

1. **Episodic memories**: `orama-episodic-index.json`
   - Configuration: `episodicEnabled`, `episodicMaxTokens`, `episodicMinSimilarity`
   - Stores general project knowledge and context

2. **Thinking memories**: `orama-thinking-index.json`
   - Configuration: `thinkingEnabled`, `thinkingMaxTokens`, `thinkingMinSimilarity`
   - Stores Claude's reasoning paired with outputs

Both indexes use Orama (pure JavaScript) for semantic search with cosine distance scoring. Each can be enabled/disabled independently through configuration, allowing fine-grained control over which memory types are retrieved.

**Important**: When resetting memories, both indexes should be cleared and rebuilt from transcripts.
