Two new environment variables control which memory systems are active:

- `LOCAL_RECALL_EPISODIC_ENABLED` (default: `false`) - Controls episodic memory search, session-start injection, and extraction
- `LOCAL_RECALL_THINKING_ENABLED` (default: `true`) - Controls thinking memory search and extraction

These defaults mean thinking memories are enabled by default while episodic memories are disabled unless explicitly enabled via environment variable. This allows independent control of each memory system.

Implementation details:
- Config schema updated in `src/core/types.ts:118-119` with `episodicEnabled` and `thinkingEnabled` boolean fields
- Environment variable parsing added in `src/utils/config.ts:56-61`
- All hooks (`user-prompt-submit.ts`, `user-prompt-submit-thinking.ts`, `session-start.ts`) updated to import and use `getConfig()` to access these flags
- MCP server (`src/mcp-server/server.ts`) updated to respect these flags for extraction/processing
