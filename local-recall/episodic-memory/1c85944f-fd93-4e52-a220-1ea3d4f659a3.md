Claude stores projects in ~/.claude/projects using a path-to-dashes naming convention. For example, `/Users/joe/Code/Syntessera/local-recall` becomes `-Users-joe-Code-Syntessera-local-recall`. The `findClaudeProjectDir` function should:

1. Convert the current working directory path to this format (replacing `/` with `-`)
2. Check for the folder in ~/.claude/projects
3. Look for transcripts directly in the project folder, NOT in a `transcripts` subfolder

This is the primary method - fallback scanning should only occur if the main lookup fails. The actual folder structure shows `.jsonl` files are stored directly in the project directory.
