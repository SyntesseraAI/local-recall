Memory files are structured markdown with YAML frontmatter containing:
- `id`: UUID
- `subject`: Brief description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, area:<name>)
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 prefix (16 chars) for deduplication

The `occurred_at` and `content_hash` fields are critical for deduplication - finding duplicate memories prevents information redundancy.
