Each memory file contains:
- `id`: UUID identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for deduplication/sorting
- `content_hash`: SHA-256 prefix (16 chars) for deduplication

Memories are idempotent - creating a duplicate returns the existing memory rather than creating a new one.
