# MCP Server Background Daemon

The MCP server runs a background daemon that processes Claude transcripts asynchronously:

1. **Sync Location**: Syncs transcripts from `~/.claude/projects/<project>/transcripts/`
2. **Processing**: Uses `claude -p` to extract memories from transcript content
3. **Change Detection**: Tracks processed transcripts with content hashes to detect changes
4. **Memory Management**: Deletes and recreates memories when transcripts change
5. **Interval**: Runs every 5 minutes

This means the Stop hook is disabled in favor of the background daemon approach for better scalability and to avoid blocking the user experience during transcript processing.
