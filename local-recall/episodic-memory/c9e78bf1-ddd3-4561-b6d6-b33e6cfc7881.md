# Transcript Collector Implementation

## Key Responsibilities

The transcript collector is responsible for:
1. Reading JSONL transcript files from Claude's cache
2. Parsing transcript events to extract memories
3. Condensing long transcripts into actionable memory items
4. Tracking processed transcripts via content hashes

## Transcript Format

Transcripts use JSONL format where each line is a JSON event object. Events contain:
- Event type (e.g., 'user-message', 'assistant-message', 'tool-call')
- Content/text payload
- Metadata like timestamps

## Memory Extraction Strategy

When processing transcripts:
1. Parse JSONL events sequentially
2. Identify significant moments (problems solved, patterns discovered, decisions made)
3. Condense multiple related events into focused memory items
4. Include specific details: file paths, function names, error messages
5. Use appropriate scope (global vs file-specific vs area)
