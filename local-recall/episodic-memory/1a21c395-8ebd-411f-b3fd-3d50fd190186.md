Memory files are markdown files with YAML frontmatter containing:
- `id`: UUID unique identifier
- `subject`: Brief one-line description (max 200 chars)
- `keywords`: Array of searchable keywords (lowercase, specific)
- `applies_to`: Scope string (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for when the event occurred
- `content_hash`: SHA-256 hash prefix (16 chars) of the content

Memories are stored in `local-recall/episodic-memory/` and are version-controlled in git.
