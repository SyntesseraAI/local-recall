# Transcript JSONL Format

Each line in a Claude Code transcript JSONL file is a JSON object representing a message in the conversation.

## Message Structure

Each message object contains:
- `role`: Either `"user"` or `"assistant"`
- `content`: The message text content
- Additional metadata fields (varies by message type)

## Parsing Approach

When reading transcripts:
1. Read the file line by line
2. Parse each line as a separate JSON object
3. Collect messages into an array
4. Filter or process based on role and content

## Important Notes

- Each line is a complete, valid JSON object - not a JSON array
- Messages preserve conversation order (chronological)
- Empty or malformed lines should be handled gracefully
- The transcript grows as the session progresses
