# MCP Server Transcript Collection

The MCP server has a background daemon that periodically:

1. Syncs transcripts from `~/.claude/projects/<encoded-path>/transcripts/`
2. Parses JSONL transcript files (each line is a JSON event)
3. Tracks processed transcripts using content hashes for change detection
4. Extracts memories from transcripts using Claude Haiku
5. Manages the memory lifecycle - deletes and recreates memories when transcripts change
6. Runs every 5 minutes (configured interval)

This allows automatic memory extraction without manual user intervention, and ensures the memory system stays in sync with actual session activity.
