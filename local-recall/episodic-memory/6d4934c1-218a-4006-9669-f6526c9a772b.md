The applies_to field determines memory scope: 'global' applies to entire codebase, 'file:<path>' is specific to a file (e.g., file:src/utils/config.ts), and 'area:<name>' relates to components/areas (e.g., area:authentication). This affects memory relevance during searches and session loading.
