All memory files are markdown with YAML frontmatter. The `applies_to` field specifies scope:
- `global` - Applies to entire codebase
- `file:<path>` - Specific to a file (e.g., `file:src/utils/config.ts`)
- `area:<name>` - Related to a component (e.g., `area:authentication`)

Other fields: id (UUID), subject (one-line), keywords (array), occurred_at (ISO-8601), content_hash (SHA-256 prefix).
