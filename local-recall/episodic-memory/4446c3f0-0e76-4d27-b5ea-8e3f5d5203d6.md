The memory system prevents duplicate memories using a two-factor approach:

1. **Content hash**: SHA-256 hash (16-char prefix) of the memory content
2. **Timestamp**: `occurred_at` ISO-8601 timestamp of when the event occurred

Before creating a new memory, the system checks if a memory with the same `occurred_at` and `content_hash` already exists. If found, it returns the existing memory ID instead of creating a duplicate.

This approach allows identical memories from different sessions while preventing true duplicates.
