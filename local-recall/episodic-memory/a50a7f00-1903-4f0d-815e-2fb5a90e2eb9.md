# MCP Server Transcript Processing

The MCP server daemon processes Claude Code transcripts to automatically extract and create memories.

## Current Implementation

The background daemon:
1. Syncs transcripts from `~/.claude/projects/<project>/transcripts/`
2. Reads JSONL transcript files
3. Processes transcripts using Claude Haiku to extract memory-worthy information
4. Creates memory files in `local-recall/episodic-memory/`
5. Tracks processed transcripts with content hashes to detect changes
6. Runs every 5 minutes

## Key Points

- Transcripts contain both user prompts and assistant responses
- Memory extraction uses Claude Haiku for efficiency
- Content hashing prevents duplicate memory creation
- The system handles transcript updates and deletions
- JSONL format allows streaming processing of large transcripts
