Each memory is a markdown file with YAML frontmatter containing metadata:
- `id`: UUID
- `subject`: Brief description
- `keywords`: Array of searchable terms
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp
- `content_hash`: SHA-256 prefix (16 chars)

Memory content follows the frontmatter. These files are version-controlled in git unlike the index.json which is gitignored.
