Each memory file is a markdown document with YAML frontmatter containing:
- `id`: Unique identifier (UUID)
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (`global`, `file:<path>`, or `area:<name>`)
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

Memory files are version-controlled and stored in `local-recall/episodic-memory/` directory.
