Each memory file is stored as a structured markdown file with YAML frontmatter containing:
- `id` - Unique identifier (UUID)
- `subject` - Brief one-line description
- `keywords` - Array of searchable keywords
- `applies_to` - Scope: `global`, `file:<path>`, or `area:<name>`
- `occurred_at` - ISO-8601 timestamp (when original event occurred)
- `content_hash` - SHA-256 hash prefix of content (16 chars, for deduplication)

Memory files should be atomic (one concept per file) and stored in `local-recall/episodic-memory/` directory.
