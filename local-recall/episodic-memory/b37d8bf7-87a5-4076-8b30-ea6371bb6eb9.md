# JSONL Transcript Format

Claude Code transcripts use JSONL format with the following key fields:
- `timestamp`: ISO-8601 format for when the message occurred
- `type`: Message type (e.g., 'user', 'assistant')
- `content`: The actual message content
- `session_id`: Links messages to a session

Each line is a separate JSON object representing one message in the conversation.

## Processing Notes
When parsing transcripts for memory extraction:
1. Parse as JSONL (one JSON object per line)
2. Extract key conversation turns
3. Condense verbose tool output
4. Preserve user intent and assistant responses
