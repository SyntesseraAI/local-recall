# Transcript Condensing for Memory Extraction

The MCP server background daemon needs to condense raw JSONL transcripts before passing them to Claude for memory extraction.

## Current State

Raw transcripts contain verbose event data that needs to be converted into a more usable format for Claude's memory extraction process.

## Required Functionality

Implement transcript condensing that converts JSONL transcript events into a structured format showing:
- `[User]` - User messages and requests
- `[Assistant]` - Claude's responses
- `[Tool: Name]` - Tool invocations (Read, Edit, Bash, Grep, etc.)
- `[Result: OK/ERROR]` - Tool invocation outcomes

## Purpose

This condensed format makes it easier for Claude to identify valuable memories without processing verbose metadata and intermediate states from the raw transcript.

## Integration

The condensing logic should be in the transcript-collector module and used by the MCP server daemon before passing transcripts to Claude for memory extraction.
