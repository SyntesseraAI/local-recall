Each memory markdown file uses YAML frontmatter to store metadata:
- `id` - Unique identifier (UUID)
- `subject` - Brief one-line description
- `keywords` - Array of searchable keywords
- `applies_to` - Scope: global, file:<path>, or area:<name>
- `occurred_at` - ISO-8601 timestamp for when the original event occurred
- `content_hash` - SHA-256 hash prefix (16 chars) of content for deduplication

Followed by markdown content containing the actual memory.
