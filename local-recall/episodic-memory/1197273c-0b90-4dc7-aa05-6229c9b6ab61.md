Memory files are stored as markdown with YAML frontmatter containing:
- `id`: Unique UUID identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for deduplication
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

Memories are idempotent - creating a duplicate returns the existing memory instead of creating a new one.
