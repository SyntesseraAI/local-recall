# Transcript Condensing Implementation

The transcript-collector component now uses transcript condensing to extract meaningful events from raw transcript JSONL files.

## Process

1. Parses JSONL transcript files from Claude's cache
2. Extracts key events: user messages, assistant responses, tool invocations, and results
3. Condenses multi-line content into single-line summaries
4. Filters out noise like acknowledgments and timestamps

## Format

Condensed events follow the pattern:
- `[User] - <message summary>`
- `[Assistant] - <response summary>`
- `[Tool: <name>] - <invocation details>`
- `[Result: OK/ERROR] - <outcome>`

## Benefits

- Reduces token count when processing transcripts
- Makes it easier to identify key interactions
- Improves context window efficiency
- Better baseline for memory extraction prompts
