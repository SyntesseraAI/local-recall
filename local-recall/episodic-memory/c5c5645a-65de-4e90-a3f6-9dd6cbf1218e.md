The MCP server daemon maintains separate processing flags:
- `isEpisodicProcessing` (boolean, default: false) - prevents concurrent episodic memory extraction runs
- `isThinkingProcessing` (boolean, default: false) - prevents concurrent thinking memory extraction runs

Both can be true simultaneously, allowing episodic and thinking extraction to run in parallel without blocking each other. This prevents duplicate processing while maximizing throughput of the daemon loop that runs every 5 minutes.
