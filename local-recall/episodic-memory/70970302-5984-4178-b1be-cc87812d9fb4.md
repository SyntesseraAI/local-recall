Each memory is stored as a markdown file with YAML frontmatter containing:
- `id`: Unique UUID identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope indicator (`global`, `file:<path>`, or `area:<name>`)
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 hash prefix (16 chars) for identifying duplicates

This structure ensures memories are atomic, searchable, and deduplication-aware.
