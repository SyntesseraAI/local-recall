# Transcript Condensing Format

When multiple lines of content need to be stored in JSONL format (one JSON object per line), the transcript condensing process must handle newlines carefully.

## Approach

Instead of storing raw newlines which break JSONL format, content with multiple lines should be:
1. Condensed into a single line representation
2. Using newline escape sequences or indicators
3. Or using a prefix-suffix pattern to denote multi-line blocks

## Example

A multi-line code snippet or message in a transcript should be stored as a single line in the JSONL file, with internal structure preserved through escaping or structured field handling.

## Usage in Memory Extraction

When the MCP server's transcript processor reads JSONL transcripts, it must handle this condensed format to properly reconstruct the original multi-line content for memory extraction.
