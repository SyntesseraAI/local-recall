# Memory Frontmatter Schema Changes

## Removed Fields
- `updated_at` - no longer needed for immutable architecture
- `UpdateMemoryInput` type - memory updates are no longer supported
- `timeWindow` configuration - no longer needed as entire history is processed

## Added Fields
- `occurred_at` - ISO-8601 timestamp of when the original event/conversation happened
- `content_hash` - SHA-256 hash prefix (16 chars) of memory content

## Updated Types
- `CreateMemoryInput` now includes `occurred_at`
- Memory frontmatter schema reflects immutable nature of memories
