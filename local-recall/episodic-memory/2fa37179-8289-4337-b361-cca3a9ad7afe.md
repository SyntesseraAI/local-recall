Each memory file includes YAML frontmatter with fields:
- `id`: UUID for unique identification
- `subject`: Brief one-line description
- `keywords`: Array of searchable terms for fuzzy matching
- `applies_to`: Scope indicator (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for chronological tracking
- `content_hash`: SHA-256 prefix for deduplication

These fields enable efficient searching and prevent duplicate memories.
