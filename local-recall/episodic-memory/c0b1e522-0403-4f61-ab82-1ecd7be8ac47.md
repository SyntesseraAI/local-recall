Memory files stored as markdown with YAML frontmatter including:
- `id`: UUID identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:/path, or area:name)
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 prefix for duplicate detection

Memories are idempotent - duplicate detection uses occurred_at + content_hash combination. Memory files ARE version-controlled and committed to git.
