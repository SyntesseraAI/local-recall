# Transcript Condensing Process

The `transcript-collector.ts` file implements transcript parsing and condensing for memory extraction.

## Key Functionality

1. **JSONL Parsing**: Reads `.jsonl` files from Claude's transcript cache
2. **Event Condensing**: Compresses transcript into condensed format showing:
   - `[User]` events - user messages/requests
   - `[Assistant]` events - Claude's responses
   - `[Tool: Name]` events - tool invocations
   - `[Result: OK/ERROR]` events - tool outcomes

## Condensed Format Structure

The condensed transcript is a text format that:
- Removes verbose details while preserving key information
- Shows tool usage and results
- Maintains conversation flow
- Is suitable for processing by memory extraction (via Claude Haiku)

## Integration

This condensed format is used by:
- MCP server memory extraction process
- Hook-based memory creation
- Background transcript processing daemon
