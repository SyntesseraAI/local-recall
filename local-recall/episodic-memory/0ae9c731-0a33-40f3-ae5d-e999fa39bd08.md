The codebase uses a configuration system via `getConfig()` from `src/utils/config.ts` that allows independent control of episodic and thinking memory features.

**Key configuration flags:**
- `episodicEnabled` - Controls whether episodic memories are retrieved and injected
- `thinkingEnabled` - Controls whether thinking memories are retrieved and injected
- Each type can be independently enabled/disabled
- Loaded via environment variables or `.local-recall.json` configuration file

**Usage pattern:**
The session-start hook checks these flags before retrieving respective memory types, allowing users to configure which memory types they want to use in their sessions.
