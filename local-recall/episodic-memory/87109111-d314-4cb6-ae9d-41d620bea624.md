Local Recall follows the standard Claude Code marketplace plugin structure (matching code-farm pattern):

- `.claude-plugin/` directory at project root containing:
  - `plugin.json` - Plugin metadata and configuration
  - `marketplace.json` - Marketplace definition with name `syntesseraai-local-recall`
- `local-recall-plugin/` at root level (not nested) containing:
  - `config/hooks.json` - Hook configurations
  - `.mcp.json` - MCP server configuration
  - `scripts/` - Bundled compiled hooks and MCP server output

The old nested `dev-marketplace/local-recall-plugin/` structure has been replaced with this flatter, standard layout.
