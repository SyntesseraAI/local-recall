Transcript files use JSONL format where each line is a JSON object representing a single event. Tool invocations include serialized Tool result objects with `type`, `status`, `error`, and potentially `output` fields. When parsing transcripts, deserialize these Tool result objects using `JSON.parse()` before processing. This affects how the transcript collector and memory extractor handle tool output.
