# Memory Extraction Process

Memories are extracted from Claude Code session transcripts using the `claude -p` command to process condensed transcript content.

## Process

1. Transcripts are collected from `~/.claude/projects/<project>/transcripts/`
2. Transcript content is condensed into a structured format with events:
   - `[User]` - User requests
   - `[Assistant]` - Assistant responses
   - `[Tool: Name]` - Tool invocations
   - `[Result: OK/ERROR]` - Outcomes
3. Condensed transcript is passed to `claude -p` for analysis
4. Claude extracts valuable memories in JSON format with metadata

## Memory Metadata

Each extracted memory includes:
- `subject`: One-line description
- `keywords`: Searchable tags
- `applies_to`: Scope (global/file/area)
- `content`: Detailed markdown content
- `occurred_at`: ISO-8601 timestamp
- `content_hash`: SHA-256 prefix for deduplication
