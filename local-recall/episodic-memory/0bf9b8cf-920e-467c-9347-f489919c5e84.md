# Memory Extraction Transcript Format

The memory extraction process expects transcripts condensed into events format:

## Event Types

- `[User]` - User requests or questions
- `[Assistant]` - Claude responses or explanations
- `[Tool: Name]` - Tool invocations (e.g., `[Tool: Read]`, `[Tool: Bash]`)
- `[Result: OK/ERROR]` - Outcome of tool calls

## Usage in Memory Extraction

This event-based format allows the memory extraction system to:
1. Understand the session flow
2. Extract relevant context around discoveries
3. Link memories to specific decisions or problems
4. Identify patterns across sessions

## Current Status

The transcript collector (`src/utils/transcript.ts`) needs to be updated to generate this condensed format instead of raw JSONL data.
