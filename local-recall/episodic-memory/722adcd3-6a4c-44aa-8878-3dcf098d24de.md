# Memory File Format

Memory files are stored as markdown with YAML frontmatter in `local-recall/episodic-memory/` directory.

## Field Definitions

- `id`: Unique UUID identifier for the memory
- `subject`: One-line brief description (max 200 chars)
- `keywords`: Array of searchable keywords for fuzzy matching
- `applies_to`: Scope indicator - `global`, `file:<path>`, or `area:<name>`
- `occurred_at`: ISO-8601 timestamp of when the original event occurred
- `content_hash`: SHA-256 hash prefix (16 chars) of the content for deduplication

## File Naming

Memory files use the UUID as filename with `.md` extension.

## Content Hash Purpose

The `content_hash` field is used to detect duplicate memories by checking if the same content (same `occurred_at` and same hash) already exists in the memory store.
