SessionStart hook loads 5 most recent memories on session begin (file-based, no vector store to avoid slow initialization). UserPromptSubmit hook performs semantic search using vector embeddings. Stop hook is disabled; instead an MCP background daemon processes transcripts every 5 minutes using `claude -p` to extract memories. All hooks use `${CLAUDE_PLUGIN_ROOT}` for relative paths.
