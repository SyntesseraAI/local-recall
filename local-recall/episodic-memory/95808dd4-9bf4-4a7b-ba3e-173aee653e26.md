The Local Recall memory system uses a dual-file structure:

1. **index.json** - Central metadata file at `/local-recall/index.json` that likely tracks all memories
2. **Individual markdown files** - Stored in `/local-recall/memories/` with UUID filenames (e.g., `d451a49a-f08e-480f-b7cd-27373f49ced9.md`)

Memory files contain conversation content with message role prefixes (`**user**:` and `**assistant**:`). The system appears designed for semantic search and long-term retention of session transcripts.
