# Hook Input Structure

Claude Code hooks receive JSON input via stdin that includes:

- `session_id` - Unique identifier for the current session
- `transcript_path` - Path to the current session's transcript
- `cwd` - Current working directory
- `prompt` - (UserPromptSubmit hook only) The user's prompt text

## Usage in local-recall

The hooks in `src/hooks/session-start.ts` and `src/hooks/user-prompt-submit.ts` parse this JSON to:
1. Determine the project context (from `cwd`)
2. Access the current transcript (from `transcript_path`)
3. Load or search memories relevant to the current session

The `session_id` can be useful for tracking which hook invocations belong to the same Claude Code session.
