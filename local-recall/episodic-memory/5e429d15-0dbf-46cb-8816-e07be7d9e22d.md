Memories use three scope levels via the `applies_to` field:

- `global` - Applies to entire codebase (architecture, conventions, preferences)
- `file:<path>` - Specific to a file (e.g., `file:src/utils/config.ts`)
- `area:<name>` - Related to a component/area (e.g., `area:authentication`, `area:database`)

Choosing the correct scope helps future assistants find relevant memories when working in specific contexts.
