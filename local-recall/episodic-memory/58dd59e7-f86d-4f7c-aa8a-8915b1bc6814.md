Local Recall stores memories as structured markdown files in `local-recall/episodic-memory/` directory. Each memory file has YAML frontmatter with fields:
- `id`: UUID
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp
- `content_hash`: SHA-256 hash prefix (16 chars)

Memory files are version-controlled in git and can be shared across team members. The markdown content after the frontmatter contains the actual memory details, code snippets, architectural decisions, bug fixes, etc.
