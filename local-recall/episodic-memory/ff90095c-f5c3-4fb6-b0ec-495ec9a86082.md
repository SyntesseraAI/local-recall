Transcripts are condensed into event format before memory extraction:
- `[User]` - User requests and questions
- `[Assistant]` - Claude's responses and explanations
- `[Tool: Name]` - Tool invocations (Read, Edit, Write, Bash, Grep, etc.)
- `[Result: OK/ERROR]` - Tool outcomes

This condensed format preserves semantic meaning while reducing token count, making it efficient for processing by Claude Haiku.
