# Session State Management

Claude Code provides contextual information through hooks that includes:

## Hook Input Parameters

- `session_id`: Unique identifier for the current Claude Code session
- `transcript_path`: Path to the transcript file being built for this session
- `cwd`: Current working directory of the user's project

## Usage Pattern

1. SessionStart hook receives these parameters and initializes context
2. UserPromptSubmit hook receives the same parameters plus the `prompt` text
3. Stop hook receives parameters to finalize the session

## Important Notes

- The `cwd` parameter allows hooks to operate on the user's project directory
- The `session_id` enables tracking memories across a single session
- The `transcript_path` points to the active transcript being built in real-time during the session
- These are passed via JSON to stdin of the hook scripts
