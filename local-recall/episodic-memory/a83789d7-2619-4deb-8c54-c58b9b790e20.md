# Transcript JSONL Format

Claude stores transcripts as JSONL (JSON Lines) files where each line is a complete JSON object representing a transcript event.

## Metadata Fields

Each transcript line contains:
- `cwd` - Current working directory where the user was working
- `session_id` - Unique identifier for the Claude Code session
- `transcript_path` - Path to the transcript file itself
- Message content and other contextual information

## Use Case for Local Recall

This metadata is essential for:
1. **Scope determination** - Using `cwd` to determine which project/workspace the memory applies to
2. **Deduplication** - Using `session_id` and timestamps to avoid duplicate memory creation
3. **Context reconstruction** - Determining the project context when memories are created

The transcript collector needs to parse JSONL correctly and extract these metadata fields to properly scope memories to the correct `applies_to` value.
