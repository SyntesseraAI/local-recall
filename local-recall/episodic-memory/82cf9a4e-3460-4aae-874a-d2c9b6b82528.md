# Transcript Message Format

Claude's transcripts are stored in JSONL (JSON Lines) format, where each line is a complete JSON message object.

## Message Structure

Each message object contains:
- `role` - Either "user" or "assistant"
- `content` - The message text content
- `timestamp` or similar timestamp fields for when the message occurred

The JSONL format allows for efficient streaming and line-by-line processing of transcripts without needing to parse the entire file into memory.

This is critical for the `transcript-collector.ts` implementation - it needs to handle this format correctly when reading and parsing transcripts for memory extraction.
