Each memory file is markdown with YAML frontmatter containing:
- `id`: Unique identifier (UUID)
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope - `global`, `file:<path>`, or `area:<name>`
- `occurred_at`: ISO-8601 timestamp for sorting and deduplication
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

Memories are idempotent - duplicate detection uses `occurredAt` and `contentHash` combination.
