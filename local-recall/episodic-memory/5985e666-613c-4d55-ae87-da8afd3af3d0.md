The `memory-extractor.ts` module handles the core logic for analyzing session transcripts and automatically generating memory files. This is a critical component of the Local Recall system's ability to capture and store institutional knowledge.

The extractor:
- Takes session transcripts as input
- Uses Claude Haiku for intelligent analysis
- Generates structured memory objects
- Deduplicates memories using content hashes
- Returns memories in the standard Local Recall format

This automation reduces manual memory creation burden and ensures consistent capture of learning across sessions.
