Each extracted memory must include:
- `subject`: Brief one-line description (max 200 chars)
- `keywords`: Array of 1-10 searchable keywords (lowercase, specific)
- `applies_to`: Scope as `global`, `file:<path>`, or `area:<name>`
- `content`: Detailed content in markdown format

Scope guidance:
- `global`: Applies to entire codebase (architecture, conventions)
- `file:<path>`: Specific to a particular file
- `area:<name>`: Related to a component or area
