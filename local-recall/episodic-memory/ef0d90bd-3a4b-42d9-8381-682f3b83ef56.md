# Session Context in Hooks

Claude Code hooks receive JSON input via stdin containing:
- `session_id`: Unique identifier for the current session
- `transcript_path`: Path to the current session's transcript file
- `cwd`: Current working directory of the user's project
- `prompt`: (UserPromptSubmit hook only) The user's submitted prompt

## Important Considerations

1. The `transcript_path` points to the JSONL file, not individual messages
2. The hooks receive this context before processing and should use it to provide relevant memories
3. The transcript path is absolute and may be on any system (macOS, Linux, Windows)
4. The `cwd` parameter should be used to determine scope for memory searches
