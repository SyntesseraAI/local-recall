# Keywords for Memory Search

Keywords are critical for making memories discoverable. They should be:

- **Specific** - Use exact terms, not generic words
- **Lowercase** - All keywords should be lowercase
- **Relevant** - Include terms that future searches might use
- **Limited** - Keep to 1-10 keywords per memory
- **Actionable** - Include problem/solution keywords, not just nouns

## Examples

**Good keywords:**
- `transcript-collector`, `file-path`, `macos`, `jsonl`
- `memory-extraction`, `actionable`, `condensing`

**Bad keywords:**
- `thing`, `stuff`, `information` (too generic)
- `ThisIsAKeyword` (should be lowercase)
- `the`, `and`, `or` (stop words)
