The MCP server background daemon runs every 5 minutes and processes:

### Episodic Memory Processing
- **Config Flag**: `LOCAL_RECALL_EPISODIC_ENABLED` (defaults to `false`)
- **Default State**: Disabled unless explicitly enabled
- **Function**: `processEpisodicTranscripts()` in daemon loop

### Thinking Memory Processing  
- **Config Flag**: `LOCAL_RECALL_THINKING_ENABLED` (defaults to `true`)
- **Default State**: Enabled by default
- **Function**: `processThinkingMemories()` in daemon loop

The daemon respects configuration in `src/utils/config.ts` which reads from environment variables. Each memory type can be independently controlled via `LOCAL_RECALL_*_ENABLED` environment variables.

Currently, thinking memory extraction runs in parallel (20 parallel tasks) while episodic processing is sequential when enabled.
