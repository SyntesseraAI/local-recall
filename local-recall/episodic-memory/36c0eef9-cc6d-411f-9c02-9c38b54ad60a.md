# MCP Server Background Daemon

The MCP server includes a background daemon that:

1. **Runs periodically**: Every 5 minutes, the daemon wakes up to process transcripts
2. **Syncs transcripts**: Reads transcripts from Claude's cache (`~/.claude/projects/<project>/transcripts/`)
3. **Extracts memories**: Uses `claude -p` (piping transcript through Claude) to extract memories
4. **Tracks processing**: Uses content hashes to detect when transcripts have changed
5. **Deduplicates**: Deletes and recreates memories when transcripts change to avoid duplicates

This allows memories to be extracted asynchronously without blocking Claude's interactive session.

## Stop Hook Status

The Stop hook is currently disabled because memory extraction is handled by the MCP server daemon instead. The daemon provides a non-blocking way to process transcripts in the background.
