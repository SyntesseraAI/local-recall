All memory files stored in `local-recall/episodic-memory/` use YAML frontmatter with these required fields:

- `id`: UUID (must be valid UUID v4)
- `subject`: Brief one-line description (max 200 chars)
- `keywords`: Array of 1-10 searchable keywords (lowercase, domain-specific)
- `applies_to`: Scope string - one of:
  - `global` (applies to entire codebase)
  - `file:<path>` (specific file path)
  - `area:<name>` (component/area name)
- `occurred_at`: ISO-8601 timestamp (when the memory occurred)
- `content_hash`: SHA-256 prefix (16 chars) for deduplication

Followed by markdown content. This structure enables searchability and prevents duplicate memories.
