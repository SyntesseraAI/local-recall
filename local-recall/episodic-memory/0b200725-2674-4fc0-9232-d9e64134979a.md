Large transcripts exceed token limits when passed to Claude for memory extraction. The project needs a transcript condensing mechanism that summarizes and compresses transcript content while preserving key information. This is critical for the Stop hook and MCP server background daemon to process transcripts efficiently without hitting Claude's context limits.
