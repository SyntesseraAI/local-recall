# MCP Server Background Daemon

## Async Transcript Processing

The MCP server runs a background daemon process that:

1. **Runs on a 5-minute interval** - Automatically processes transcripts periodically
2. **Syncs from Claude's cache** - Reads transcripts from `~/.claude/projects/<project>/transcripts/`
3. **Extracts memories asynchronously** - Uses `claude -p` to process transcripts
4. **Tracks processed files** - Maintains a log of which transcripts have been processed
5. **Handles changes** - Deletes and recreates memories when transcript content changes

This provides an alternative to the Stop hook for memory extraction, useful for cases where the Stop hook may not run or for processing transcripts from other sessions.
