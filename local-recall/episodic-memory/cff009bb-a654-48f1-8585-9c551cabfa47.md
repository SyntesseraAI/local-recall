Memory extraction analyzes Claude Code session transcripts to identify valuable insights for future assistants. Memories should be specific, concise, and actionable. Each memory requires: subject (one-line description), keywords (1-10 lowercase terms), applies_to scope (global/file/area), and markdown content. Extract architectural decisions, bug fixes, code patterns, configuration quirks, and component relationships. Avoid generic programming knowledge, temporary debug steps, and sensitive data.
