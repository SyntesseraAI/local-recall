The local-recall configuration system has two functions:

1. `loadConfig(baseDir)` - Loads and parses configuration from environment variables and `.local-recall.json` file
2. `getConfig()` - Returns the currently loaded configuration object

Hooks and other modules import `getConfig` to access configuration values after the config has been loaded. This pattern is used across multiple files:
- `src/hooks/user-prompt-submit.ts`
- `src/hooks/user-prompt-submit-thinking.ts`
- `src/hooks/session-start.ts`
- `src/mcp-server/server.ts`

The config object contains both episodic and thinking memory settings (enabled flags, max tokens, similarity thresholds).
