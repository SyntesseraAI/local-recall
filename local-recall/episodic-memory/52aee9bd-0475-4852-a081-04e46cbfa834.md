Each memory file uses YAML frontmatter to store metadata with the following required fields:
- `id`: Unique identifier (UUID)
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope string (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for the original event
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

This structure enables efficient searching, filtering, and deduplication of memories across sessions.
