# Transcript Format

Claude Code stores transcripts as JSONL files (JSON Lines format) where each line is a separate JSON object representing an event in the session.

## Structure

Each event object contains:
- Event metadata (type, timestamp, etc.)
- Role information (user vs assistant)
- Content (text, tool names, results)

## Key Points

1. **File Format**: `.jsonl` extension, not `.json`
2. **One event per line**: Each line is a complete, valid JSON object
3. **Sequential events**: Events are ordered chronologically
4. **Parseable line-by-line**: Can be processed with streaming or line-by-line parsing

## Parsing Strategy

When reading transcripts:
1. Read the file line by line
2. Parse each line as a separate JSON object
3. Process events sequentially
4. Handle parsing errors gracefully (malformed JSON lines may exist)
