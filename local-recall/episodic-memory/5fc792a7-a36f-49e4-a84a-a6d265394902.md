Memory files are stored as markdown files with YAML frontmatter in `local-recall/episodic-memory/` and ARE tracked in git. The index file (`local-recall/index.json`) is gitignored and auto-generated. Memory files are atomic (one concept per memory) with fields:
- `id`: UUID
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:path, or area:name)
- `occurred_at`: ISO-8601 timestamp
- `content_hash`: SHA-256 prefix for deduplication
