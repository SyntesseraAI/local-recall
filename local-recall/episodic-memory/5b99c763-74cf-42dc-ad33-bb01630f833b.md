Added two environment variables to toggle memory systems independently:

- `LOCAL_RECALL_EPISODIC_ENABLED` (default: `false`) - Controls episodic memory search, session-start injection, and extraction/processing
- `LOCAL_RECALL_THINKING_ENABLED` (default: `true`) - Controls thinking memory search and extraction/processing

These are parsed in `src/utils/config.ts` and stored in the config schema (`src/core/types.ts`). The flags are checked in multiple places:
- Hooks: `src/hooks/user-prompt-submit.ts`, `src/hooks/user-prompt-submit-thinking.ts`, `src/hooks/session-start.ts`
- MCP server: `src/mcp-server/server.ts` (controls which extraction/processing tasks run in the daemon)

This allows users to completely disable either memory system without removing code, useful for debugging or reducing overhead.
