Each memory file in `local-recall/episodic-memory/` contains YAML frontmatter with these required fields:
- `id`: UUID unique identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for sorting and deduplication
- `content_hash`: SHA-256 prefix (16 chars) for duplicate detection

The content_hash enables idempotent memory creation - duplicate memories are detected and not recreated.
