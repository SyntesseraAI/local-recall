Claude Code transcripts have messages with `content` as an array of content blocks, not a simple string. Each block has a `type` field:
- `type: "thinking"` with `thinking` field (internal reasoning)
- `type: "text"` with `text` field (assistant output)
- `type: "tool_use"` / `type: "tool_result"` for tool interactions

The actual format also uses `type` field for message role classification (not `role`), though some older/legacy formats may still use `role`.
