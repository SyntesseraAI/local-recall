Memory files use YAML frontmatter to store structured metadata:
- `id`: UUID for unique identification
- `subject`: Brief one-line description (max 200 chars)
- `keywords`: Array of 1-10 searchable keywords (lowercase, specific)
- `applies_to`: Scope string (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp of when the original event occurred
- `content_hash`: SHA-256 prefix (16 chars) for deduplication

The `occurred_at` and `content_hash` fields are used by MemoryExtractor.findDuplicate() to prevent storing duplicate memories across sessions.
