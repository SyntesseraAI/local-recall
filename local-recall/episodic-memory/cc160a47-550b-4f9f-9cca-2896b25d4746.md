Claude Code transcript files are stored in JSONL (JSON Lines) format where each line represents a separate event.

**Format Structure**:
- Each line is a complete JSON object
- Lines contain events with: type (User/Assistant/Tool), content, timestamp, metadata
- User input and Claude's responses are separate lines
- Tool invocations and results are tracked separately

**Location**: Transcripts are typically stored at `~/.claude/projects/<project-id>/transcripts/`

**Usage in Local Recall**:
- The MCP server daemon syncs and processes transcripts
- parseTranscriptForMemories() function reads and parses this format
- Content hash tracking enables change detection for deduplication
