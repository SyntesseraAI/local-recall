# Transcript Condensing with Claude API

The transcript-collector uses Claude's API to condense lengthy transcripts before memory extraction. This improves efficiency and reduces token usage.

## Process

1. Reads raw JSONL transcripts from `~/.claude/projects/<project>/transcripts/`
2. Extracts messages from the JSONL format
3. Sends transcript text to Claude Haiku via API (`claude -p --model haiku`)
4. Claude condenses the transcript, removing redundancy while preserving key information
5. Condensed transcript is then processed for memory extraction

## Benefits

- Reduces token usage by condensing verbose transcripts
- Preserves important context needed for memory extraction
- Makes memory extraction more accurate by focusing on key events
- Reduces storage overhead for memory files

## Implementation Location

Implemented in `src/core/transcript-collector.ts` as part of the memory extraction pipeline.
