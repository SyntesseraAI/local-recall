Each memory file follows a structured format with YAML frontmatter containing metadata (id, subject, keywords, applies_to, occurred_at, content_hash) followed by markdown content. This hybrid format allows both programmatic access to metadata and human-readable content. Memory files are version-controlled in git and stored in local-recall/episodic-memory/.
