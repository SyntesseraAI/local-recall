## Claude Projects Directory Location

Claude Code stores project transcripts in a specific directory structure:

- **Base path**: `~/.claude/projects/`
- **Project folder naming**: Uses the full project path with special character encoding (spaces as `_`, forward slashes as `-`)
- **Transcript file location**: `<project-folder>/.claude/projects/<encoded-path>/transcripts/`
- **File format**: JSONL files with `.jsonl` extension, one message per line
- **File naming**: Based on session identifier or timestamp

## Key Details

- The `~/.claude/projects/` directory is where Claude Code caches all session data
- Project paths are encoded in the folder names to handle special characters
- Each transcript is a JSONL file where each line is a complete JSON message object
- This structure is used by the transcript-collector to find and process session transcripts

## Usage in Local Recall

The transcript-collector in `src/core/transcript-collector.ts` uses this path structure to:
1. Find the user's project directory
2. Locate the encoded project folder in `~/.claude/projects/`
3. Read JSONL transcript files from the transcripts subdirectory
4. Parse and process messages for memory extraction
