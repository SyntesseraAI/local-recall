# Transcript Structure (JSONL Format)

Each transcript file in Claude's cache is in JSONL format (one JSON object per line). Each object represents a message in the conversation.

## Object Structure

Typical fields in each JSON object:
- `role`: The speaker role (e.g., "user", "assistant")
- `content`: The message content (string or array of content blocks)
- Metadata about timestamps, tool usage, etc.

## Processing Considerations

When parsing transcripts for memory extraction, iterate through each JSONL line and deserialize individual JSON objects. Handle both simple string content and complex content arrays (for tool invocations, images, etc.).
