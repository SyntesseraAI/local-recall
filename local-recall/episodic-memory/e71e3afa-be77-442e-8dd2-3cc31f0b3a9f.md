When Ollama returns EOF errors on embedding requests, check the input size first before assuming server issues. Ollama's embedded model subprocess may panic when receiving inputs that exceed the model's context window.

The error path shows Ollama's internal communication: `Post "http://127.0.0.1:PORT/embedding": EOF` indicates the model subprocess crashed, not the main Ollama API. Always validate input constraints match model capabilities.
