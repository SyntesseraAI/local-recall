# Transcript Condenser Purpose

The `transcript-condenser.ts` module prepares transcript data for memory extraction by:

1. **Aggregating transcript context**: Combines relevant messages and events from one or more transcript files
2. **Creating condensed format**: Produces a structured, condensed representation of the session
3. **Preparing input for extraction**: Formats the data in a way that's optimal for `claude -p` memory extraction

## Usage in Memory Extraction Pipeline

The condensed transcript output is used as input to the `memory-extractor.ts` module, which then:
- Uses `claude -p` to process the condensed transcript
- Extracts structured memories based on the condensed content
- Creates memory markdown files

## Design Pattern

This follows a two-stage pipeline:
1. **Stage 1 (Condenser)**: Raw transcript → Condensed format
2. **Stage 2 (Extractor)**: Condensed format → Extracted memories
