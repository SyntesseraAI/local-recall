Memory files are stored as markdown with YAML frontmatter containing:
- `id`: UUID unique identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope - `global`, `file:<path>`, or `area:<component>`
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication
- Content follows frontmatter in markdown format

Memories are atomic (one concept per file) and version-controlled in git.
