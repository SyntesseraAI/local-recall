Each memory is a markdown file with YAML frontmatter containing:
- `id`: UUID identifier
- `subject`: One-line description (max 200 chars)
- `keywords`: Array of searchable keywords (lowercase, specific)
- `applies_to`: Scope - `global`, `file:<path>`, or `area:<name>`
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

The content section can include code snippets, architectural decisions, bug fixes, user preferences, and project conventions.
