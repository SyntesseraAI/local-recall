# Vector Search Engine Migration

The project transitioned from SQLite with sqlite-vec extension to Orama (pure JavaScript search engine).

## Key Changes
- **Old approach**: Native SQLite with vector extension for embeddings
- **New approach**: Orama as pure JavaScript vector store
- **Benefit**: Eliminates native dependencies, simplifies deployment
- **Files removed**: `http-server.ts`, `daemon-client.ts`, `database.ts`

## Impact
This enables hooks to perform vector searches directly without needing a separate daemon process or HTTP communication layer.
