The project now stores episodic and thinking memories in JSONL format instead of separate markdown files. This provides better performance and structure.

**Old Format**:
- Each memory = one markdown file with YAML frontmatter
- Stored in `local-recall/episodic-memory/*.md` and `local-recall/thinking-memory/*.md`
- Read entire file to access metadata

**New Format**:
- Multiple memories per JSONL file (`.jsonl` extension)
- Each line is a complete JSON record with all metadata
- Faster sequential reads and metadata queries
- More efficient for bulk operations

**Implementation**:
- `src/core/episodic-jsonl-store.ts` - JSONL storage for episodic memories
- `src/core/thinking-jsonl-store.ts` - JSONL storage for thinking memories
- Old markdown files are preserved for backwards compatibility
- New code uses JSONL stores exclusively

**Migration Strategy**: The codebase maintains dual-read capability during transition period - can read from both old markdown files and new JSONL stores.
