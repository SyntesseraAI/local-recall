Memory files are markdown with YAML frontmatter containing metadata. Key fields: `id` (UUID), `subject`, `keywords`, `applies_to` (scope), `occurred_at` (ISO-8601 timestamp), `content_hash` (SHA-256 prefix, 16 chars). Deduplication uses `occurred_at` timestamp and `content_hash` to detect duplicates. Memory files are version-controlled in git.
