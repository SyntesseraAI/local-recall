Memory files use structured YAML frontmatter with these fields:

- **id**: UUID, unique identifier
- **subject**: One-line description (max 200 chars)
- **keywords**: Array of searchable keywords (lowercase, specific)
- **applies_to**: Scope indicator - one of:
  - `global` - Applies to entire codebase
  - `file:<path>` - Specific to a file (e.g., `file:src/utils/config.ts`)
  - `area:<name>` - Related to component/area (e.g., `area:authentication`)
- **occurred_at**: ISO-8601 timestamp for chronological sorting and deduplication
- **content_hash**: SHA-256 prefix (16 chars) of content for deduplication

**Deduplication**: `findDuplicate(occurredAt, contentHash)` checks for existing duplicates before creating new memories.
