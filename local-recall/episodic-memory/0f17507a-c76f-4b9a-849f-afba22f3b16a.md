The MCP server runs a background daemon that:
1. Monitors Claude's transcript cache at `~/.claude/projects/<project>/transcripts/`
2. Uses content hashing to detect transcript changes
3. Sends transcripts to `claude -p` (Claude Haiku) to extract memories
4. Creates memories from extracted content using `memory_create` tool
5. Tracks processed transcripts to avoid reprocessing
6. Runs every 5 minutes

This asynchronous approach avoids blocking the Stop hook.
