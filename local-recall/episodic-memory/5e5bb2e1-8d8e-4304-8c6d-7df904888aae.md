When committing plugin and memory updates:

1. Use `git status` to identify all modified files in the plugin directory
2. Stage changes with `git add dev-marketplace/local-recall-plugin/...`
3. Create descriptive commits that list the specific changes:
   - Version bumps
   - New skills added
   - Tool description improvements
   - Metadata updates
   - New memory files
4. Push to main branch with `git push`

The process ensures all plugin changes and associated memory files are tracked together in version control.
