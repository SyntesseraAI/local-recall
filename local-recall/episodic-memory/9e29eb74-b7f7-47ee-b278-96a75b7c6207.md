Each memory file in `local-recall/episodic-memory/` follows a structured format with required frontmatter fields:
- `id`: UUID
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope indicator (global, file:/path/to/file, area:component-name)
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 prefix (16 chars) for duplicate detection

Memories are idempotent - the system checks for duplicates using occurred_at and content_hash before creating new memories.
