The `applies_to` field in memory metadata determines scope:

1. **global** - Applies to entire codebase (architecture, conventions, preferences)
2. **file:<path>** - Specific to a particular file (e.g., `file:src/utils/config.ts`)
3. **area:<name>** - Related to a component or area (e.g., `area:authentication`, `area:database`)

This allows memories to be targeted appropriately when loading during SessionStart or when used for semantic search context.
