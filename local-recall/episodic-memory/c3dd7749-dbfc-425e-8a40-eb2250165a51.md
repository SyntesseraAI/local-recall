Each memory file in `local-recall/episodic-memory/` contains:
- `id`: UUID identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords for fuzzy matching
- `applies_to`: Scope ('global', 'file:<path>', or 'area:<name>')
- `occurred_at`: ISO-8601 timestamp for sorting and deduplication
- `content_hash`: SHA-256 hash prefix (16 chars) to detect duplicate content

Memories are idempotent - creating a duplicate returns the existing memory based on `occurred_at` and `content_hash` matching.
