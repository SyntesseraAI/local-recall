# Transcript Collection from Claude Cache

The `transcript-collector.ts` module syncs transcripts directly from Claude's cache at `~/.claude/projects/`.

## Implementation Details

- Reads from the default Claude projects directory on macOS
- Converts working directory paths to Claude's folder naming convention
- Processes JSONL transcript files
- Used by MCP server daemon for background memory extraction

## Why This Approach

This allows the Local Recall system to work with Claude Code's native transcript storage without requiring users to configure custom paths or worry about transcript management. The daemon can run independently and automatically extract memories from any Claude Code session.

## Important

Make sure the transcript collector handles the platform-specific path conversions correctly when dealing with different working directories.
