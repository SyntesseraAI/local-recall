# JSONL Transcript Format

Claude Code transcripts are stored in JSONL format (newline-delimited JSON), where each line is a complete JSON object representing a single event.

## Event Types

Transcript events can include:
- User messages
- Assistant responses  
- Tool invocations (Read, Edit, Bash, etc.)
- Tool results (OK/ERROR outcomes)
- Session metadata

Each event is a complete, valid JSON object on its own line.

## Key Considerations

- No JSON array wrapper - just individual objects separated by newlines
- Each object must be valid JSON and parseable independently
- This format allows streaming and incremental parsing
- Use `readline` or similar line-based readers to parse JSONL files
