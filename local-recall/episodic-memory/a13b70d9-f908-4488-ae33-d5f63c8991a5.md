When updating the local-recall plugin, bump the version number in both:
1. `/package.json` (main project version)
2. `/dev-marketplace/local-recall-plugin/.claude-plugin/plugin.json` (plugin metadata)

Incrementing the version number forces Claude's plugin system to deploy a fresh copy to the cache directory with a versioned path like `~/.claude/plugins/cache/local-recall-marketplace/local-recall/0.1.1/`. This invalidates any cached old versions and ensures all Claude instances use the updated plugin.

Other Claude instances can then reinstall from the marketplace to get the new bundled version.
