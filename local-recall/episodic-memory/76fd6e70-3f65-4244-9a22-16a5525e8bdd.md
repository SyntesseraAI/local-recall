Created comprehensive TypeScript types for parsing Claude Code transcript JSONL files in `src/types/transcript-schema.ts`. The schema covers:

**Entry Types:**
- `UserEntry` - User prompts with session context
- `AssistantEntry` - Claude responses with thinking blocks, content blocks
- `SystemEntry` - System messages and status updates
- `FileHistorySnapshot` - File state snapshots for change detection
- `QueueOperation` - Tool invocations and results (Bash, Read, Write, Grep, LSP, etc.)

**Content Block Types:**
- `TextBlock` - Plain text responses
- `ToolUseBlock` - Tool invocations with parameters
- `ToolResultBlock` - Tool execution results

**Common Fields:**
- `sessionId` - Session identifier
- `timestamp` - ISO-8601 timestamp
- `type` - Entry type discriminator

Use these types to safely parse and extract data from transcripts without string manipulation.
