# Transcript Condenser Tests

Comprehensive test suite in `tests/unit/core/transcript-condenser.test.ts` validates:

## Test Coverage
- Parsing each JSONL entry type (user, assistant, system, etc.)
- Extracting minimal but sufficient content from each type
- Handling edge cases:
  - Empty content blocks
  - Missing optional fields
  - Large outputs (verified they're truncated)
  - Thinking blocks with multiple entries
  - Tool operations with various result types

## Test Patterns
Uses Vitest for unit testing with fixtures for different transcript scenarios. All tests pass successfully.

## Integration
Tests ensure the condenser produces valid input for the memory extraction subprocess.
