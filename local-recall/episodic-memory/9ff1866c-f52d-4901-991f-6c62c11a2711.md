Each memory file is markdown with YAML frontmatter containing:
- `id`: Unique UUID identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:/path, or area:name)
- `occurred_at`: ISO-8601 timestamp for deduplication/sorting
- `content_hash`: SHA-256 hash prefix (16 chars) for detecting duplicates

Memory files are version-controlled in git, unlike the gitignored `index.json` and `recall.log`.
