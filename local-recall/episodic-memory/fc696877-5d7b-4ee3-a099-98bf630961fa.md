EpisodicJsonlStore extends JsonlStore with episodic-specific entry types. Files use prefix 'episodic-' followed by 6-digit padding. Stores add/delete/embedding entries for memories. Each memory add entry includes id, subject, keywords, applies_to, occurred_at, content_hash. Embedding entries store 768-dimensional vectors separately. Files are loaded in sequential order (000001 → 000002 → ...) during initialization.
