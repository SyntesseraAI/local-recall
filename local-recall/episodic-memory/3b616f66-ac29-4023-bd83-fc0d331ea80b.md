When adding the episodic/thinking enabled flags, the following files all needed to be updated consistently:

1. `src/core/types.ts` - Config interface schema
2. `src/utils/config.ts` - Environment variable parsing
3. `src/hooks/user-prompt-submit.ts` - Episodic memory search
4. `src/hooks/user-prompt-submit-thinking.ts` - Thinking memory search  
5. `src/hooks/session-start.ts` - Session startup memory injection
6. `src/mcp-server/server.ts` - Server-side extraction and tools

All files needed the same pattern: import `getConfig`, check the appropriate enabled flag before processing memories. This suggests configuration changes should be tracked broadly across the codebase to ensure consistency.
