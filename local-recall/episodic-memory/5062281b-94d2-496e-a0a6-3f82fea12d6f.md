Memory files follow a structured format with YAML frontmatter including: `id` (UUID), `subject` (brief description), `keywords` (searchable array), `applies_to` (scope: global/file:/path/area:name), `occurred_at` (ISO-8601 timestamp), and `content_hash` (SHA-256 prefix). The memory content goes in the markdown body below the frontmatter. Each memory should be atomic - one concept per file.
