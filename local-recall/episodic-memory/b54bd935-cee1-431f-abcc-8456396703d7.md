# Memory Index Storage and Gitignore

The memory index is stored at `local-recall/index.json` and is explicitly gitignored.

## Index Characteristics

- **Location**: `local-recall/index.json`
- **Format**: JSON object mapping keywords to memory IDs
- **Gitignore Status**: YES - included in `.gitignore`
- **Generation**: Auto-generated by running `npm run build` or via the `index_rebuild` MCP tool
- **Purpose**: Fast keyword lookup for search operations

## Why Gitignore?

The index is a cache that's:
- Derived from source memory files (in `local-recall/episodic-memory/`)
- Can be regenerated anytime
- Specific to each local installation
- Not needed in version control

## Rebuilding

The index should be rebuilt:
- After adding/removing memories
- Before deploying or sharing
- Via the `index_rebuild` MCP tool
- Or by running the build script
