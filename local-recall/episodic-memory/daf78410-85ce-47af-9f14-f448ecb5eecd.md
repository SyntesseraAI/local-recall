Each memory file is a structured markdown document with YAML frontmatter containing:
- `id` - Unique identifier (UUID)
- `subject` - Brief one-line description
- `keywords` - Array of searchable keywords
- `applies_to` - Scope: `global`, `file:<path>`, or `area:<name>`
- `occurred_at` - ISO-8601 timestamp (for deduplication and sorting)
- `content_hash` - SHA-256 hash prefix of content (16 chars, for deduplication)

Memories are idempotent - duplicate memories (same occurred_at and content_hash) are not created.
