Memories are structured YAML frontmatter + markdown files stored in `local-recall/episodic-memory/`. Each memory includes: id (UUID), subject (brief description), keywords (searchable terms), applies_to (scope), occurred_at (ISO-8601 timestamp), and content_hash (SHA-256 prefix). This structure enables fuzzy searching and semantic retrieval of architectural decisions, bug fixes, and conventions specific to the project. Memories should be atomic (one concept per file) and version-controlled in git.
