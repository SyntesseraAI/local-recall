# Transcript Structure

Claude Code transcripts contain sequential events that document the conversation and tool usage:

## Event Types

1. **User Message Events**
   - Type: User message content
   - Contains the full text of user prompts

2. **Assistant Message Events**
   - Type: Assistant response content
   - Contains Claude's explanations, reasoning, and responses

3. **Tool Invocation Events**
   - Type: Tool calls (Read, Edit, Write, Bash, Grep, etc.)
   - Includes tool name and parameters

4. **Result Events**
   - Type: Tool results (OK/ERROR)
   - Contains output or error information

## Memory Extraction Approach

The transcript contains all information needed to create memories:
- User requests and context
- Solutions implemented
- Technical decisions made
- Problems encountered and resolved
- Code patterns and conventions discovered
