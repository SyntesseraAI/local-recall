# Transcript Condensing Implementation

The `src/core/transcript-collector.ts` file is being refactored to implement transcript condensing functionality.

## Current State

The file has modifications (as seen in git status) that implement:
- Reading and parsing JSONL transcript files
- Condensing transcripts to improve memory extraction efficiency
- Converting detailed transcript events into a summarized format

## Implementation Details

The transcript collector:
1. Reads JSONL files from Claude's transcript cache
2. Parses each line as a separate JSON object
3. Condenses the transcript to extract key information for memory creation
4. Handles transcript format: `[User]`, `[Assistant]`, `[Tool: Name]`, `[Result: OK/ERROR]`

## Why This Matters

Transcript condensing:
- Reduces token usage when passing transcripts to Claude for memory extraction
- Improves efficiency of the memory creation process
- Enables better memory extraction from long sessions
- Reduces noise in extracted memories by focusing on key events
