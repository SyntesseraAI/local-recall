# Memory Extraction Workflow

The local-recall system uses a structured JSON format for internal memory extraction communication between the transcript analysis system and the memory creation system.

## Format

Memories are extracted as a JSON object with a `memories` array. Each memory object contains:
- `subject`: Brief one-line description (max 200 chars)
- `keywords`: Array of 1-10 searchable keywords (lowercase, specific)
- `applies_to`: Scope string (`global`, `file:<path>`, or `area:<name>`)
- `content`: Detailed memory content in markdown format

## Return Format

Always return ONLY a valid JSON object with no explanation, markdown formatting, or code blocks - just raw JSON. If no valuable memories can be extracted, return: `{ "memories": [] }`

## Key Principle

This JSON-based approach allows the transcript analysis to be deterministic and easily parseable by downstream tools.
