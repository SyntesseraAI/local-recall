Each memory is stored as a structured markdown file with YAML frontmatter:

Required fields:
- `id` - UUID unique identifier
- `subject` - Brief one-line description
- `keywords` - Array of searchable keywords
- `applies_to` - Scope: `global`, `file:<path>`, or `area:<name>`
- `occurred_at` - ISO-8601 timestamp for deduplication and sorting
- `content_hash` - SHA-256 hash prefix (16 chars) of content for deduplication

Memory files are version-controlled (tracked in git) while index.json and recall.log are gitignored. Memories are idempotent - no updates, only create/read/delete operations.
