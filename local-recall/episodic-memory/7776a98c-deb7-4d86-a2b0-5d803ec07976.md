Local Recall now maintains TWO separate vector stores and indices:

1. **Episodic Memories** (`orama-episodic-index.json`)
   - Traditional semantic memories about codebase
   - Generated from transcripts via memory extraction
   - Injected on UserPromptSubmit via `user-prompt-submit.ts` hook

2. **Thinking Memories** (`orama-thinking-index.json`)
   - Claude's internal reasoning paired with outputs
   - Extracted from thinking blocks in transcripts
   - Shows "how I thought â†’ what I produced"
   - Injected on UserPromptSubmit via `user-prompt-submit.ts` hook

**Configuration**: Both have independent settings:
- `episodicEnabled/thinkingEnabled` - Toggle each type
- `episodicMaxTokens/thinkingMaxTokens` - Budget per type
- `episodicMinSimilarity/thinkingMinSimilarity` - Threshold per type

**Classes**:
- `ThinkingVectorStore` - Manages thinking memory embeddings
- `ThinkingExtractor` - Extracts thinking blocks (20 parallel) from transcripts
- `ThinkingProcessedLog` - Tracks processed transcripts for thinking extraction
