The MCP SDK updated from ^1.0.0 to 1.23.0 with breaking changes:

- `Server` class constructor now requires `options` parameter with `name` field
- `StdioServerTransport` now takes `stdin` and `stdout` stream objects instead of being auto-initialized
- Old pattern: `new Server({ ... })` → New pattern: `new Server({ name: '...', ... })`
- Old pattern: `new StdioServerTransport()` → New pattern: `new StdioServerTransport({ stdin, stdout })`

Fix applied in src/mcp-server/server.ts to create transport with proper stream parameters. This ensures MCP server starts correctly with newer SDK versions.
