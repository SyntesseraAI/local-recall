# Using Read Tool for Transcript Files

When accessing transcript files, the user prefers using the Read tool instead of Bash `cat` commands. This aligns with the project's tooling preferences:

- **Read tool**: Use for file reading operations (properly handles permissions and is optimized)
- **Bash**: Reserve for actual terminal/system commands

This preference ensures consistency with the codebase guidelines and better tool utilization.
