Memory files in local-recall follow a consistent format:

1. **YAML Frontmatter** with fields:
   - `id`: UUID for unique identification
   - `subject`: One-line brief description (max 200 chars)
   - `keywords`: Array of searchable keywords (lowercase, 1-10 items)
   - `applies_to`: Scope indicator (global, file:<path>, or area:<name>)
   - `occurred_at`: ISO-8601 timestamp for deduplication and sorting
   - `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

2. **Markdown Content**: Actual memory details in markdown format

3. **Deduplication**: Uses occurred_at timestamp and content_hash to prevent duplicate memories when same transcript is processed multiple times

4. **Storage**: All memories are version-controlled and stored in local-recall/episodic-memory/ directory
