The transcript collector now enforces UUID filename validation as a core filtering mechanism. This is a defensive pattern that ensures robustness when processing transcripts:

- **Pattern**: `UUID_REGEX = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}\.jsonl$/i`
- **Helper function**: `isUuidFilename(filename: string): boolean`
- **Applied consistently** across all transcript discovery and processing methods to maintain data integrity
- **Prevents issues** with stray or misconfigured files being treated as valid transcripts
