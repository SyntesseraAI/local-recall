Memory files are stored in `local-recall/episodic-memory/` as markdown with YAML frontmatter containing:
- `id`: UUID for unique identification
- `subject`: One-line brief description
- `keywords`: Array of searchable keywords (lowercase)
- `applies_to`: Scope (`global`, `file:<path>`, or `area:<name>`)
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

Memories are idempotent - duplicates detected via occurrence timestamp and content hash are not created.
