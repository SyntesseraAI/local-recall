# MCP Server Daemon Processing

The MCP server runs a background daemon that:

1. **Syncs transcripts** from Claude's cache at `~/.claude/projects/<project>/transcripts/`
2. **Processes transcripts** using `claude -p` to extract memories via Claude's processing API
3. **Tracks processed transcripts** with content hashes for change detection
4. **Deletes and recreates memories** when transcripts change
5. **Runs periodically** every 5 minutes

This asynchronous approach allows memory extraction without blocking the Stop hook execution. The daemon independently manages transcript-to-memory conversion and deduplication.
