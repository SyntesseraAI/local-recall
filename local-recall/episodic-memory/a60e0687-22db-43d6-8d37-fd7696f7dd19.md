# Transcript JSONL Format Structure

Claude's transcript JSONL files contain complete session data as JSON objects, one per line.

## Key Fields in Each Line

- `type`: Event type (e.g., "tool_call", "tool_result", "user_message", "assistant_message")
- `content`: The actual message or tool data
- `tool_name`: Name of the tool invoked (for tool_call type)
- `tool_input`: Parameters passed to the tool
- `timestamp`: When the event occurred

## Important Characteristics

- Each line is a complete, valid JSON object
- Must be parsed line-by-line, not as a single JSON array
- Contains the full history of a Claude session
- Includes all tool invocations and results
- Used for memory extraction in the MCP server background daemon
