Each memory file is stored as markdown with YAML frontmatter containing:

- `id`: Unique identifier (UUID)
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords (lowercase, specific)
- `applies_to`: Scope indicator (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp of when the original event occurred
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

The `occurred_at` and `content_hash` fields enable duplicate detection and content change tracking across sessions, preventing duplicate memory creation when the same information is rediscovered.
