Memory files are markdown files with YAML frontmatter containing:
- `id`: UUID identifier
- `subject`: One-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp
- `content_hash`: SHA-256 hash prefix (16 chars)

Memories are stored in `local-recall/episodic-memory/` directory and are version-controlled in git. Thinking memories go in `local-recall/thinking-memory/`.
