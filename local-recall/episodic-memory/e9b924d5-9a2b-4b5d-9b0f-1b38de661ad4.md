# Transcript Condensing for Memory Extraction

The project implements transcript condensing to compress verbose transcripts into compact event summaries before memory extraction.

## Purpose

Condensed transcripts reduce token usage and improve clarity when extracting memories from full session transcripts.

## Format

Condensed transcripts use a simplified event format:
- `[User]` - User requests or questions
- `[Assistant]` - Claude's responses and explanations
- `[Tool: Name]` - Tool invocations (Read, Edit, Write, Bash, Grep, etc.)
- `[Result: OK/ERROR]` - Outcome of tool invocations

This condensed format is more efficient for memory extraction than raw transcript data.
