Each memory is a markdown file with YAML frontmatter containing: id (UUID), subject (one-line description), keywords (array for search), applies_to (scope: global|file:<path>|area:<name>), occurred_at (ISO-8601 timestamp), content_hash (SHA-256 prefix). Content section contains actual memory in markdown. Memory files are version-controlled and tracked in git.
