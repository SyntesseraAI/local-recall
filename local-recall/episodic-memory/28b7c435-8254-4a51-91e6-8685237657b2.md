Memory files use markdown with YAML frontmatter. Required fields:

- `id`: UUID (unique identifier)
- `subject`: One-line brief description (max 200 chars)
- `keywords`: Array of searchable keywords for discovery
- `applies_to`: Scope - 'global' (entire codebase), 'file:/path/to/file' (specific file), or 'area:component-name' (component area)
- `occurred_at`: ISO-8601 timestamp for sorting and deduplication
- `content_hash`: SHA-256 hash prefix (16 chars) of content for deduplication detection

Memories are idempotent - createMemory returns existing memory if duplicate detected. No update operation exists; memories are created or deleted.
