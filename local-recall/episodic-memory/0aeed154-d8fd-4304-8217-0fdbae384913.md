# Transcript Memory Extraction Pipeline

The system uses Claude Haiku (via `claude -p` command) to extract memories from session transcripts.

## Process

1. Transcripts are collected from Claude's cache at `~/.claude/projects/`
2. Transcript condensing reduces raw events to key interactions
3. Claude Haiku analyzes condensed transcripts to extract memories
4. Memories are created via the MCP server's `memory_create` tool

## Implementation

- MCP server runs as a background daemon
- Processes transcripts asynchronously every 5 minutes
- Tracks processed transcripts with content hashes for change detection
- Deletes and recreates memories when transcripts change

## Command Reference

Memory extraction command: `claude -p --model haiku --instructions '[memory extraction instructions]' < [condensed transcript]`

This design allows automated knowledge capture without manual memory creation.
