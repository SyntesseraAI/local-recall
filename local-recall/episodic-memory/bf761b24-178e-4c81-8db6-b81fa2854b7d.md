# Transcript JSON Structure

Claude Code transcripts contain detailed event information:

## Event Types

1. **User Messages** - Contains the full user input
2. **Assistant Messages** - Contains Claude's responses
3. **Tool Invocations** - Records of tool calls with:
   - Tool name (Bash, Read, Edit, Grep, etc.)
   - Input parameters
   - Full output/results
   - Execution status

## Important State Information

Transcripts capture:
- File contents at the time of Read operations
- Exact code changes from Edit/Write operations
- Bash command outputs (stdout/stderr)
- Tool execution timestamps
- The full state of work performed

## For Memory Extraction

This rich transcript data allows extracting:
- What files were modified and how
- What bugs were discovered and fixed
- What architectural decisions were made
- What user preferences are evident
- What problems were solved and their solutions

The tool invocations provide concrete evidence and context for memory creation.
