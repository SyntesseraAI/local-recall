# Transcript Collector Flexibility

The transcript collector (`src/core/transcript-collector.ts`) is designed to handle both:

1. **Real transcripts**: Actual `.jsonl` files from Claude Code's transcript cache
2. **Synthetic transcripts**: Array of transcript events for testing and development

## Implementation Details

The collector accepts either:
- A file path to a `.jsonl` transcript file
- An array of transcript event objects

This flexibility allows for:
- Unit testing with mock data
- Development without needing real Claude sessions
- Integration with the MCP server daemon that processes actual transcripts

## Benefits

This design pattern makes it easier to test transcript processing logic independently of the Claude environment.
