The background daemon in the MCP server processes transcripts asynchronously using Claude Haiku (`claude -p --model haiku`) to extract memories. The `memory-extractor.ts` module handles transcript condensing to manage token limits when sending transcripts to Claude for analysis. This approach allows memories to be created from sessions without blocking the user.
