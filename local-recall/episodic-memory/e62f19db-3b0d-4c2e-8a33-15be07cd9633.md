# DS_Store File Filtering in Transcript Collection

When collecting transcript files from `~/.claude/projects/*/transcripts/`, the code must filter out macOS system files, specifically `.DS_Store`.

Without filtering, the transcript collector will attempt to parse `.DS_Store` as a JSONL file, causing parsing errors. This is a critical filtering requirement for the `transcript-collector.ts` implementation on macOS systems.

Recommended solution: Filter files by extension (`.jsonl`) or by name pattern before attempting to parse.
