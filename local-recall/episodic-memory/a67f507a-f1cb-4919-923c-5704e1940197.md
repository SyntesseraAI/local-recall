Two environment variables control memory system activation:

- `LOCAL_RECALL_EPISODIC_ENABLED` - Defaults to `false`, controls episodic memory search, session-start injection, and extraction
- `LOCAL_RECALL_THINKING_ENABLED` - Defaults to `true`, controls thinking memory search and extraction

These flags allow independent control of each memory system. When disabled, the system skips:
- Search operations in user-prompt-submit hooks
- Memory injection during session-start
- Transcript processing and extraction for that memory type

Implemented in `src/core/types.ts` (config schema), `src/utils/config.ts` (parsing), and applied to hooks in `src/hooks/user-prompt-submit.ts`, `src/hooks/user-prompt-submit-thinking.ts`, `src/hooks/session-start.ts`, and MCP server in `src/mcp-server/server.ts`.
