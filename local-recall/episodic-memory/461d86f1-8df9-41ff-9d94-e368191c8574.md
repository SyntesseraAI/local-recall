Memory files are stored as markdown files in `local-recall/episodic-memory/` and are tracked in git (version-controlled). Each file has YAML frontmatter containing:
- `id`: UUID identifier
- `subject`: Brief description
- `keywords`: Array of searchable terms
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp
- `content_hash`: SHA-256 prefix for deduplication

The actual content is stored in markdown format below the frontmatter.
