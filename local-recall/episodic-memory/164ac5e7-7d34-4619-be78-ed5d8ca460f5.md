# JSONL Transcript Format

Claude's transcript files use JSONL (JSON Lines) format, not regular JSON arrays.

## Format Details

- Each line in the file is a separate, complete JSON object
- Lines are separated by newlines (`\n`)
- NOT a JSON array (no opening `[` or closing `]`)
- Each object typically represents a message or event in the conversation

## Implementation Considerations

When parsing transcripts:
1. Read the file line by line (don't try to parse as a single JSON object)
2. Parse each line individually as JSON
3. Handle empty lines gracefully
4. Implement proper error handling for malformed lines
5. Collect all valid parsed objects into an array for processing

## Why This Format

JSONL is used for streaming/incremental data because:
- Easy to append new messages without re-parsing the entire file
- Can process line-by-line without loading entire file into memory
- Standard format for log files and event streams
