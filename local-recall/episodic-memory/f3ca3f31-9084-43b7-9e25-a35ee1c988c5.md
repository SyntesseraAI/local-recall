Each memory markdown file requires frontmatter with: id (UUID), subject (brief description), keywords (array for search), applies_to (scope: global/file:/path/to/file/area:component-name), occurred_at (ISO-8601 timestamp for deduplication/sorting), and content_hash (SHA-256 prefix for deduplication). Memory files are idempotent and tracked in git, while the SQLite database (memory.sqlite) and embedding cache are gitignored.
