The Local Recall project extracts memories from Claude Code session transcripts by leveraging Claude's haiku model. The process is:

1. MCP server or background daemon monitors transcripts in `~/.claude/projects/<project>/transcripts/`
2. Reads transcript content and detects changes via content_hash
3. Invokes `memory-extractor.ts` to process the transcript
4. Uses Claude haiku (via `claude -p --model haiku`) to analyze and extract memories based on specific prompts
5. Creates memory files in `local-recall/episodic-memory/` with extracted data
6. Updates deduplication tracking to prevent duplicate memory creation

This enables asynchronous, intelligent memory extraction without blocking Claude's real-time interaction.
