The processed-log.jsonl files track which transcripts have been processed to enable idempotent memory extraction:

- Each entry: `{"path": "...", "contentHash": "sha256-prefix", "processedAt": "iso-timestamp"}`
- Content hash is SHA-256 prefix (16 chars) to detect when transcripts change
- When transcript content changes, old memories are deleted and new ones created
- Files can have millions of entries, requiring streaming/line-by-line reads
- JSONL format chosen over SQLite for version-control compatibility and simple line-based appends
