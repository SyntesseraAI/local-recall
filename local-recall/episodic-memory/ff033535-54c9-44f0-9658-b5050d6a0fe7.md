# Vector Store Architecture Migration

Local Recall migrated from JSON-based Orama indexes to SQLite with sqlite-vec extension for vector storage. The key architectural changes:

- **Storage**: `local-recall/memory.db` (SQLite database)
- **No more**: `orama-episodic-index.json` or `orama-thinking-index.json` files
- **Vector operations**: Uses sqlite-vec for fast vector search via `embedding MATCH` syntax
- **Embedding generation**: Still uses Ollama with `nomic-embed-text` model (768 dimensions)

This provides better performance, persistence, and atomic operations compared to JSON indexes.
