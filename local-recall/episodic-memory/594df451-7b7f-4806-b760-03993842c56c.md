Each memory file in `local-recall/episodic-memory/` uses YAML frontmatter with these key fields:
- `id`: UUID
- `subject`: Brief description
- `keywords`: Searchable terms
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for deduplication and recency sorting
- `content_hash`: SHA-256 prefix (16 chars) for duplicate detection

The `occurred_at` and `content_hash` fields are used by `findDuplicate()` to prevent storing duplicate memories and ensure fresh memories are ranked higher than stale ones when search scores are equal.
