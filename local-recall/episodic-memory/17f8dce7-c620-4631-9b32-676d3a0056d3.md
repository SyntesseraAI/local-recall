The MCP server implements a background daemon that:
- Processes transcripts asynchronously every 5 minutes
- Syncs transcripts from Claude's cache
- Uses content hashes to detect changes
- Deletes and recreates memories when transcripts change
- Handles transcript deduplication

This asynchronous approach prevents blocking the main server while ensuring all transcripts are processed regularly.
