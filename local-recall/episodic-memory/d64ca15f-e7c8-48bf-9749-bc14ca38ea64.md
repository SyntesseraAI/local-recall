Local Recall enables persistent memory storage across Claude Code sessions through structured markdown files with YAML frontmatter. Key architectural components:

- **Core**: Memory manager (CRUD ops), vector store (SQLite + embeddings), semantic search
- **Hooks**: SessionStart (loads recent memories), UserPromptSubmit (semantic search)
- **MCP Server**: Exposes memory tools, runs background daemon for transcript processing
- **Storage**: Episodic memory files in `local-recall/episodic-memory/` (git-tracked), SQLite database in `local-recall/memory.sqlite` (git-ignored)

Memory files use YAML frontmatter with fields: id, subject, keywords, applies_to, occurred_at, content_hash.
