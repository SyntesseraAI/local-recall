Memory files in Local Recall are stored in `local-recall/episodic-memory/` directory as markdown files with YAML frontmatter. Each memory contains: `id` (UUID), `subject` (brief description), `keywords` (searchable array), `applies_to` (scope: global, file:<path>, or area:<name>), `occurred_at` (ISO-8601 timestamp), `content_hash` (SHA-256 prefix for deduplication). The episodic-memory files ARE version-controlled and will be committed to git. The SQLite database (`memory.sqlite`) and log file (`recall.log`) are gitignored. This design allows memories to be shared among team members while maintaining automatic database management.
