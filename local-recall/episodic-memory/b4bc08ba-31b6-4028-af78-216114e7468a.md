Memory files in Local Recall are stored as structured markdown with YAML frontmatter containing metadata:
- `id`: Unique UUID identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 hash prefix (16 chars) for duplicate detection

Memory files ARE version-controlled and will be committed to git. The SQLite database and vector embeddings are gitignored.
