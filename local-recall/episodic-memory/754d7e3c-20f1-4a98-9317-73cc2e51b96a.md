The MCP server includes a background daemon that automatically processes transcripts:

- **Frequency**: Runs every 5 minutes
- **Source**: Syncs transcripts from Claude's cache (`~/.claude/projects/<project>/transcripts/`)
- **Processing**: Uses `claude -p` to extract memories from transcripts
- **Tracking**: Uses processed-log.ts to track processed transcripts with content hashes for change detection
- **Updates**: Deletes and recreates memories when transcripts change (detected via content hash comparison)
- **Stop Hook**: Currently disabled - memory extraction delegated to MCP server daemon instead of Stop hook

This async approach avoids blocking the Claude Code session while ensuring memories stay current.
