# Transcript Condensing Implementation

The `transcript-collector.ts` file implements transcript condensing to compress transcript data before sending to Claude for memory extraction.

## Purpose

Condensing reduces the size of transcripts by removing redundant information and only keeping the essential events:
- User messages
- Assistant responses
- Tool invocations
- Tool results

## Implementation Details

The condensing process:
1. Reads JSONL transcript files
2. Filters events to keep only relevant message types
3. Compresses redundant information
4. Outputs condensed format for Claude's analysis

## Why This Matters

The MCP server daemon processes transcripts asynchronously every 5 minutes. Condensing the transcripts before sending them to Claude ensures:
- Faster processing times
- Lower API costs
- Better focus on relevant context for memory extraction
- Efficient use of Claude's context window
