The memory extractor module at `src/core/memory-extractor.ts` is responsible for:

1. **Core Functionality**:
   - Accepts transcript files and processes them to extract valuable information
   - Uses Claude's API to analyze transcripts and generate memory content
   - Creates structured memory files with proper YAML frontmatter

2. **Memory Creation Process**:
   - Calls `createMemory()` from the memory manager to persist extracted memories
   - Handles deduplication using `findDuplicate()` to prevent duplicate memories
   - Uses content hashing to identify when transcripts change

3. **Integration Points**:
   - Part of the stop hook workflow for processing completed sessions
   - Works with transcript-collector to access transcript data
   - Relies on memory extraction prompts (`src/prompts/memory-extraction.ts`)

4. **Key Behaviors**:
   - Handles transcript condensing for efficient processing
   - Uses proper error handling for API interactions
   - Generates appropriate metadata (keywords, scope, timestamps)
