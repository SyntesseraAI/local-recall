# Transcript JSONL Message Structure

Each line in a transcript JSONL file contains a message object with:
- `role`: Either "user" or "assistant"
- `content`: Varies by role and message type:
  - User messages: String with prompt text OR array of content blocks (for complex inputs with images/files)
  - Assistant messages: String with text response OR array of tool invocations

The transcript-collector needs to handle both formats when parsing messages, as user prompts may contain structured content blocks instead of plain text.
