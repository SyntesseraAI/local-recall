In the parsed transcript structure from `parseTranscript()`, different block types have different content properties:
- Regular content blocks (User, Assistant tool results): use `c.text` property
- Thinking blocks: use `c.thinking` property

This inconsistency can cause silent data loss if downstream code assumes all blocks use the same property accessor. The parsing correctly identifies thinking blocks via `<p>` tags, but content extraction must use the appropriate property for each block type.
