# Simplified Hook Architecture

Hooks no longer need to communicate with an HTTP daemon. They now perform vector search directly using Orama.

## Hook Flow
- **SessionStart**: Directly reads recent memories from files, no daemon needed
- **UserPromptSubmit**: Directly searches Orama indexes for relevant memories

## Benefits
- Eliminates process isolation complexity
- Faster memory retrieval (no HTTP overhead)
- Easier to debug and test
- No daemon startup/shutdown issues
