# Transcript Collector Implementation

The transcript collector (`src/core/transcript-collector.ts`) is responsible for processing JSONL transcript files from Claude's cache and extracting memory candidates.

## Key Functions

- **`collectTranscripts(transcriptPath)`**: Main entry point for processing transcript files
- **Processes JSONL format**: Each line is a separate JSON event object
- **Event extraction**: Analyzes user messages, assistant responses, and tool invocations
- **Memory candidates**: Identifies key information worth preserving as memories

## Integration

The transcript collector is used by:
- MCP server background daemon (processes transcripts every 5 minutes)
- Stop hook (when enabled) - processes transcripts on session end

## Important Notes

- Transcripts are in JSONL format (one JSON object per line)
- Each event contains structured information about interactions
- The collector needs to handle incomplete or malformed transcript files gracefully
- Used for automated memory extraction without user intervention
