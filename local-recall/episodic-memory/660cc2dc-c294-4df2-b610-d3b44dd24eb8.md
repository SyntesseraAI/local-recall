# Transcript Condensing for Memory Extraction

The transcript collector in `src/core/transcript-collector.ts` implements transcript condensing to extract only the most relevant information for memory creation.

## Condensing Strategy

Instead of processing raw full transcripts, the system:
1. Filters out irrelevant events (tool completions, verbose outputs)
2. Keeps user requests and assistant responses
3. Retains tool invocations (name only, not full configs)
4. Maintains action outcomes

## Implementation

The condensing logic creates a simplified timeline that Claude Haiku can process efficiently to extract memories, avoiding unnecessary token usage on verbose transcript details.
