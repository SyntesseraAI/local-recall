# Transcript Change Detection and Memory Extraction

The MCP server's background daemon processes transcripts for memory extraction using content hash-based change detection.

## How It Works

1. **Tracking**: The daemon tracks processed transcripts using content hashes to detect changes
2. **Change Detection**: When a transcript changes (different content hash):
   - Delete any existing memories associated with the old transcript
   - Process the updated transcript with Claude
   - Create new memories from the updated content
3. **Frequency**: Runs every 5 minutes

## Location

Source: `src/core/transcript-collector.ts` and MCP server implementation

## Purpose

This ensures memories stay in sync with actual session transcripts without manual intervention, and prevents duplicate memories when transcripts are updated or regenerated.
