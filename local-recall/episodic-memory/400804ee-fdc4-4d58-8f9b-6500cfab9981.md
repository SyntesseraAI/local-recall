Memory files are stored in local-recall/episodic-memory/ as markdown files with YAML frontmatter. Required metadata fields: id (UUID), subject (brief description), keywords (array), applies_to (scope), occurred_at (ISO-8601 timestamp), content_hash (SHA-256 prefix). Memory files ARE version-controlled and committed to git. The SQLite database (memory.sqlite) is gitignored and auto-generated from markdown files.
