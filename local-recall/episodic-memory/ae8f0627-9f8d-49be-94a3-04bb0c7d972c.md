The `ExtractorOptions` interface was extended to include a `concurrency` parameter (optional, defaults to 10). This allows callers to control how many transcript extractors run in parallel when processing multiple transcripts. The parameter is passed to the internal semaphore used by `processAllTranscripts` to control concurrent execution.
