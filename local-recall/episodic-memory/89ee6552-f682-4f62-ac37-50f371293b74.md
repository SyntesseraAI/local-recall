# Keyword Extraction Guidelines for Memories

When extracting keywords from prompts and transcripts:

## Avoid Generic Terms

Keywords like "code", "project", "help", "fix", "issue" are too broad and reduce search effectiveness. These terms appear in almost every transcript and make fuzzy search results noisy.

## Prefer Specific Terms

Instead extract:
- Specific tool names (e.g., "embedding", "vector-store", "fasttext")
- Component names (e.g., "memory-extractor", "transcript-collector")
- File paths and modules (e.g., "src/core", "hooks")
- Specific problems or patterns (e.g., "tokenizer-cache", "model-download")
- Technology names (e.g., "fastembed", "bge-model")

## Impact

Specific keywords:
- Improve search relevance
- Reduce false positives
- Make memories more actionable
- Help future assistants find exactly what they need

## Implementation

The `claude -p --model haiku` command used in user-prompt-submit hook can be guided to produce better keywords through careful prompt engineering.
