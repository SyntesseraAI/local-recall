# Memory Frontmatter Schema Update

## Fields Added
- `occurred_at`: ISO-8601 timestamp of when the event/conversation happened (source timestamp)
- `content_hash`: SHA-256 hash prefix (16 chars) of memory content for deduplication

## Fields Removed
- `updated_at`: No longer needed for immutable memories
- `timeWindow`: Configuration field removed (no longer using time-window-based filtering)

## Updated Types
- `CreateMemoryInput`: Now includes `occurred_at` parameter
- Removed `UpdateMemoryInput` type entirely
- Memory schema reflects immutable, append-only design
