# TypeScript Type Safety Preference

When implementing TypeScript code in this project, **avoid using the `any` type**. Instead:

1. Define proper TypeScript interfaces or types for data structures
2. Use generics where appropriate for flexible but type-safe code
3. Use union types to represent multiple possible types
4. Use `unknown` as a last resort, with type guards for narrowing

This ensures better type safety, improved IDE support, and more maintainable code across the project.
