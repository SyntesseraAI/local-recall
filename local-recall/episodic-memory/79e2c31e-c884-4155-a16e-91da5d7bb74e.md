SessionStart hook loads 5 most recent memories on session begin (full reload, no vector store). UserPromptSubmit hook uses semantic search via vector store when user submits prompt. Stop hook is currently disabled - memory extraction handled by MCP server daemon running every 5 minutes. Hooks configured in hooks.json and executed as shell commands receiving/returning JSON via stdin/stdout.
