The default value for `episodicEnabled` has been changed from `false` to `true` in the codebase. This affects:

1. **Schema definition** (`src/core/types.ts:118`): The Zod schema now defaults `episodicEnabled` to `true`
2. **Documentation** (`CLAUDE.md`): The configuration table has been updated to reflect the new default

Both episodic and thinking memories are now enabled by default without requiring explicit configuration. This means that unless a user explicitly disables episodic memory retrieval via environment variables or `.local-recall.json`, the system will retrieve and inject episodic memories during sessions.
