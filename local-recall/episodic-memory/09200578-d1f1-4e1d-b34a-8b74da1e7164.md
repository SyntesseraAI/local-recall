Updated `extractMemoriesFromTranscript()` in `memory-extractor.ts` to use the new transcript condenser:

1. Added import for `TranscriptCondenser` and `CondensedTranscript` type
2. Reads raw JSONL transcript content from file
3. Calls `condenseTranscript()` to parse and filter the data
4. Passes condensed data to `buildMemoryExtractionPrompt()` instead of raw content
5. Maintains all existing memory extraction logic downstream

This creates a cleaner pipeline: Raw JSONL → Condense → Extract → Create Memories, with token-heavy JSONL processing abstracted away from the prompt generation.
