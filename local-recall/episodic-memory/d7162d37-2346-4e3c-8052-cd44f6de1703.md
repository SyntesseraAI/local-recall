# Memory File Format with YAML Frontmatter

Local Recall stores memories as markdown files with YAML frontmatter containing metadata.

## Frontmatter Fields

- `id`: Unique identifier (UUID)
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope - `global`, `file:<path>`, or `area:<name>`
- `occurred_at`: ISO-8601 timestamp
- `content_hash`: SHA-256 hash prefix for deduplication

## Important Notes

- Memory files are version-controlled and committed to git
- The `index.json` file (generated index) is gitignored
- Each memory should focus on one concept
- Keywords should be lowercase and specific for better search results
