# Transcript Processing and Memory Extraction

Transcripts are stored in JSONL format (one JSON object per line). When processing transcripts:

1. **Condensed format** - Transcripts contain events like:
   - `[User]` - User requests or questions
   - `[Assistant]` - Claude's responses
   - `[Tool: Name]` - Tool invocations with parameters
   - `[Result: OK/ERROR]` - Outcomes of tool invocations

2. **Memory extraction** - Uses Claude Haiku (`claude -p --model haiku`) to:
   - Analyze the transcript content
   - Extract valuable memories
   - Return structured memory objects

3. **Deduplication** - Uses content hashes (SHA-256 prefix) to:
   - Detect duplicate memories from same transcript
   - Prevent creating redundant memories when transcripts are reprocessed

4. **Timestamp tracking** - Uses `occurred_at` field to:
   - Track when the original event occurred
   - Support sorting and timeline ordering
   - Enable deduplication of same-event memories from different sessions
