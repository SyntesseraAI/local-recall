Memory files are stored as structured markdown with YAML frontmatter in `local-recall/episodic-memory/` and ARE tracked in git. The SQLite database (`memory.sqlite`) and log files are gitignored and auto-generated.

Each memory file contains:
- Unique UUID identifier
- Brief subject description
- Searchable keywords array
- Scope indicator (global, file:<path>, or area:<name>)
- Timestamp of when the event occurred (for deduplication and sorting)
- SHA-256 content hash prefix for deduplication
- Actual memory content in markdown format

Memories are idempotent - creating a memory with the same `occurred_at` and `content_hash` returns the existing one rather than creating a duplicate.
