# Transcript Condensing for Memory Extraction

The local-recall system uses a condensing process to extract memories from session transcripts.

## Process Overview

1. **Raw Transcript**: JSONL format with detailed event data
2. **Condensing**: Transform raw transcript into condensed format with simplified events
3. **Memory Extraction**: Use Claude Haiku to analyze condensed transcript and extract memories

## Condensed Format

The condensed format represents transcript events as:
- `[User]` - User requests or questions
- `[Assistant]` - Claude's responses or explanations
- `[Tool: Name]` - Tool invocations (Read, Edit, Write, Bash, Grep, etc.)
- `[Result: OK/ERROR]` - Tool invocation outcomes

## Benefits

- Reduces token count for analysis (important for cost and performance)
- Simplifies the structure for parsing and analysis
- Makes it easier for Claude Haiku to identify key memories
- Maintains essential context while removing verbose details

## Implementation

The condensing logic is in `src/core/transcript-collector.ts` - specifically the transcription condensing/transformation functions that convert raw JSONL transcripts into the simplified event format.
