The system maintains separate Orama vector stores:
- `orama-episodic-index.json` for episodic memories (project context, decisions, conventions)
- `orama-thinking-index.json` for thinking memories (thought + output pairs from previous sessions)

Both use Ollama embeddings (nomic-embed-text, 768 dimensions) for semantic similarity. Memories are scored using cosine distance (0.0-1.0) with recency tie-breaking when scores are equal. Minimum similarity thresholds are configurable per memory type (default 0.5).
