Each memory file is stored as markdown with YAML frontmatter containing:
- `id`: Unique UUID identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords for discovery
- `applies_to`: Scope indicator (global | file:<path> | area:<name>)
- `occurred_at`: ISO-8601 timestamp for chronological ordering and deduplication
- `content_hash`: SHA-256 hash prefix (16 chars) of content for duplicate detection

Memory deduplication checks both `occurred_at` and `content_hash` to prevent duplicates from being stored.
