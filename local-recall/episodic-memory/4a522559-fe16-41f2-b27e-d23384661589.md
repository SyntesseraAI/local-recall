Local Recall's core functionality is organized into:
- `memory.ts`: CRD operations (create, delete, read, list), including duplicate detection via `findDuplicate(occurredAt, contentHash)`
- `index.ts`: Maintains searchable keyword index stored at `local-recall/index.json` (gitignored, auto-generated)
- `search.ts`: Fuzzy search implementation with `searchByKeywords()`, `searchBySubject()`, `searchByScope()`
