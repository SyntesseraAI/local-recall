# Session Transcript Format

Claude Code stores transcripts as JSONL files where each line is a complete JSON object representing a message in the conversation.

## Transcript Structure

Each line contains:
- `role`: Either "user" or "assistant"
- `content`: The full message content (text, not truncated)
- Message metadata and context information

## Key Insight

The full conversation history is available as-is - no need for special parsing or reconstruction. This enables the MCP server to process complete transcripts and extract meaningful memories from full context.
