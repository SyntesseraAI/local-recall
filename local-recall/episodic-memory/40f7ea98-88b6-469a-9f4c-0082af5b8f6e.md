Memory files use markdown with YAML frontmatter containing: id (UUID), subject (brief description), keywords (array), applies_to (global/file:/path/area:name), occurred_at (ISO-8601), and content_hash (SHA-256 prefix). The content section contains the actual memory text with code snippets, decisions, or bug fixes. Files are stored in local-recall/episodic-memory/ and are version-controlled.
