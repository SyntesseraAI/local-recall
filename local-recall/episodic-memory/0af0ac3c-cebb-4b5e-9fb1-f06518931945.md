The logger utility in `src/utils/logger.ts` defines its interface to accept a single string argument for messages. When fixing memory extractor logging (line 197), calls like `logger.error('message', error)` would fail because the logger doesn't accept multiple parameters. Correct usage: `logger.error('message: ' + error.message)` or template strings. This is important for any code adding logging to the memory extraction pipeline.
