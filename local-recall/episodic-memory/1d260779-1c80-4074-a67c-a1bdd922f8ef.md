Episodic memory processing in the MCP server is controlled by the `LOCAL_RECALL_EPISODIC_ENABLED` environment variable. It defaults to `false` in the configuration. To enable episodic memory processing, add `LOCAL_RECALL_EPISODIC_ENABLED=true` to the `env` section of the MCP server configuration in `.claude/settings.json` or `.claude/settings.local.json`. The config is defined in `src/utils/config.ts` and read by `src/mcp-server/server.ts` which uses it to control the background daemon's `runMemoryExtraction` behavior.
