# SessionStart Hook Memory Injection

The session-start hook should automatically search for and inject relevant memories into Claude's initial context when a new session begins.

## Current Implementation Status

The session-start hook exists in the codebase but may not be fully functional for injecting memories. It should:

1. Receive session context (session_id, transcript_path, cwd)
2. Load the memory index from `local-recall/index.json`
3. Search for globally relevant memories
4. Output memory content to stdout for injection

## Expected Behavior

When a user starts a new Claude Code session in a project with local-recall installed:
- Memories from previous sessions should be automatically available
- This provides context continuity across sessions
- Memory search should consider the current project context
