Added two environment variables to control memory systems:

- `LOCAL_RECALL_EPISODIC_ENABLED` (default: `false`) - Controls episodic memory search, session-start injection, and extraction
- `LOCAL_RECALL_THINKING_ENABLED` (default: `true`) - Controls thinking memory search and extraction

Implemented in:
- `src/core/types.ts:118-119` - Added to config schema
- `src/utils/config.ts:56-61` - Added env var parsing
- `src/hooks/user-prompt-submit.ts` - Updated to check episodic flag before searching
- `src/hooks/user-prompt-submit-thinking.ts` - Updated to check thinking flag before searching
- `src/hooks/session-start.ts` - Updated to check episodic flag before injecting
- `src/mcp-server/server.ts` - Updated to check flags for tool availability

These flags allow users to selectively disable memory extraction and retrieval without removing existing memories, making it easy to switch between episodic-only, thinking-only, or both memory systems.
