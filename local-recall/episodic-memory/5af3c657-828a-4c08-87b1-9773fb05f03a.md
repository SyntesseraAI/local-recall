Local Recall stores memories as markdown files with YAML frontmatter in the `local-recall/episodic-memory/` directory. Each memory file contains:
- `id`: Unique UUID identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, area:<name>)
- `occurred_at`: ISO-8601 timestamp
- `content_hash`: SHA-256 hash prefix for deduplication

Memory files are version-controlled in git, while the SQLite database (memory.sqlite) and recall.log are gitignored.
