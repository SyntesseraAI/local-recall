Memory files stored in `local-recall/episodic-memory/` use a structured YAML frontmatter format with these required fields:

**Required Fields:**
- `id` - UUID for unique identification
- `subject` - Brief one-line description (max 200 chars)
- `keywords` - Array of searchable keywords (lowercase, 1-10 items)
- `applies_to` - Scope: `global`, `file:<path>`, or `area:<name>`
- `occurred_at` - ISO-8601 timestamp for deduplication and sorting
- `content_hash` - SHA-256 hash prefix (16 chars) of content for change detection

**Memory Organization:**
- Each memory file is markdown with YAML frontmatter
- Memory files ARE version-controlled in git
- The `index.json` file is gitignored and auto-generated
- Deduplication uses `occurred_at` + `content_hash` combination

This structure enables efficient searching, deduplication, and version control of memories.
