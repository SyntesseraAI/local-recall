Each memory is stored as a markdown file with structured YAML frontmatter containing:
- `id`: Unique identifier (UUID)
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords for discovery
- `applies_to`: Scope indicator (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for sorting and deduplication
- `content_hash`: SHA-256 prefix (16 chars) for detecting duplicate content

This structure enables fuzzy search, deduplication, and efficient memory retrieval.
