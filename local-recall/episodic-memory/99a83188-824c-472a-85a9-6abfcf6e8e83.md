Local Recall has migrated from individual markdown memory files to JSONL (JSON Lines) format for efficient storage. The system now:

- Stores episodic memories in `episodic-XXXXXX.jsonl` files with 6-digit padding
- Stores thinking memories in `thinking-XXXXXX.jsonl` files with 6-digit padding
- Uses multi-file support to organize memories across multiple JSONL files
- Individual markdown files in `episodic-memory/` and `thinking-memory/` directories contain extracted and deduplicated memories

This migration improves performance and scalability while maintaining human-readable markdown files for version control.
