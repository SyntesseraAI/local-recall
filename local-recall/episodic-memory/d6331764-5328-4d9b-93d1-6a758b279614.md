TranscriptMessage interface includes:
- role: 'user' | 'assistant'
- content: string (the main response text)
- thinking?: string (optional - Claude's reasoning/planning)
- timestamp: string (ISO-8601)

Thinking content comes from ContentBlock[] arrays where blocks can be `{type: 'thinking', thinking: string}` or `{type: 'text', text: string}`. When extracting, use different property names: thinking blocks use `.thinking`, text blocks use `.text`.
