The config system in `src/utils/config.ts` provides:

- `loadConfig()` - Main function that loads configuration from environment variables or `.local-recall.json`
- `getConfig()` - Utility function (likely for getting specific config values)

All hooks and the MCP server import both functions to access configuration. When adding new config options, they should be:
1. Added to the config schema in `src/core/types.ts`
2. Parsed in `src/utils/config.ts` with environment variable fallback
3. Used with `getConfig()` in hooks and server code
