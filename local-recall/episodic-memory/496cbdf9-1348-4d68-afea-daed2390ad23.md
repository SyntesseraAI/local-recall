# Transcript Condensing for Memory Extraction

Local Recall has implemented a transcript condensing feature that summarizes long transcripts before passing them to the memory extraction process.

## Purpose

- Reduces token usage when processing transcripts with Claude Haiku
- Maintains essential information for memory extraction while removing verbose output
- Makes the extraction process more efficient and cost-effective

## Implementation

The `transcript-collector` uses transcript condensing to prepare transcripts for the memory extraction pipeline. This allows the system to handle longer sessions without proportional increases in API token usage.

## File Location

Implementation is in `src/core/transcript-collector.ts`
