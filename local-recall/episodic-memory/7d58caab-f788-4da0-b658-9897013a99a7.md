# Transcript Message Format

Each message in a Claude Code transcript JSONL file has a specific structure:

## Message Fields

- `role`: The sender of the message ("user", "assistant")
- `content`: The message content (string or array of content blocks)
- `timestamp`: When the message was sent (ISO-8601 format)
- `metadata`: Additional information about the message

## Content Variations

For assistant messages, `content` can be:
- A simple string for text responses
- An array of content blocks including:
  - Text blocks
  - Tool use blocks (for function calls)
  - Tool result blocks

## Important for Processing

- Messages are in chronological order within a file
- Tool use sequences appear as separate content blocks within a single message
- Tool results appear in subsequent messages
- User messages typically contain the prompt text
- Assistant messages may contain reasoning, explanations, and tool invocations
