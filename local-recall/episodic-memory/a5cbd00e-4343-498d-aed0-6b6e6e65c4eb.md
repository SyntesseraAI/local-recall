All memory files are stored as markdown with YAML frontmatter containing:
- `id`: Unique identifier (UUID)
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for deduplication
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

Memory files should be atomic (one concept per file) with specific, searchable keywords. They are stored in `local-recall/episodic-memory/` and are version-controlled in git.
