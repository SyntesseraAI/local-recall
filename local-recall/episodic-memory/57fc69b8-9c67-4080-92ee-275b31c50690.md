Episodic memories document decisions, fixes, and conventions:

- YAML frontmatter metadata: `id`, `subject`, `keywords`, `applies_to`, `occurred_at`, `content_hash`
- Scope options: `global` (codebase-wide), `file:<path>` (specific file), `area:<name>` (component)
- Keywords are lowercase and specific for better search matching
- Subject is brief one-line description (max ~200 chars)
- Content uses markdown for readability
- Each memory is atomic - one concept per file
- Deduplication: Uses `occurred_at` timestamp + `content_hash` to detect duplicates
