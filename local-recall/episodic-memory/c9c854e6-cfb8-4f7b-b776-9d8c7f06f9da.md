# Memory Extraction Task

The Local Recall project is designed to extract valuable memories from Claude Code session transcripts. This is a structured memory extraction process with specific guidelines.

## Key Principles

1. **What to Extract**: Architectural decisions, bug fixes, code patterns, configuration quirks, user preferences, component relationships, performance considerations
2. **What to Avoid**: Generic programming knowledge, temporary debugging steps, obvious information, sensitive data, quickly-outdated information
3. **Specificity**: Include file paths, function names, and concrete details
4. **Scope Categorization**: Use `global`, `file:<path>`, or `area:<name>` appropriately
5. **Output Format**: ONLY valid JSON with no explanation or markdown formatting

## Memory Metadata Fields

- `subject`: Brief one-line description (max 200 chars)
- `keywords`: Array of 1-10 searchable keywords (lowercase, specific)
- `applies_to`: Scope string
- `content`: Detailed memory in markdown format
