Changed the default value of `episodicEnabled` from `true` to `false` in the config schema.

This was updated in two places:
- `src/core/types.ts:118` - The Zod schema definition
- `CLAUDE.md` - Documentation table

Users can now enable episodic memory retrieval by explicitly setting `LOCAL_RECALL_EPISODIC_ENABLED=true` (environment variable) or `"episodicEnabled": true` in `.local-recall.json` config file.

This represents a shift to opt-in behavior for episodic memory rather than opt-out.
