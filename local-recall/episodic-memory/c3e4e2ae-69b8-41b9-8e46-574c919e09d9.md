All memory files are stored as markdown with structured YAML frontmatter metadata.

**Required fields**:
- `id`: UUID for unique identification
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 hash prefix (16 chars) of content for deduplication

**Storage**: Memory files are stored in `local-recall/episodic-memory/` and ARE version-controlled (committed to git).

**Deduplication**: The system uses `occurred_at` + `content_hash` to detect and prevent duplicate memories.
