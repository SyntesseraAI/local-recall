Each memory file uses YAML frontmatter with these required fields:
- `id`: Unique identifier (UUID)
- `subject`: Brief one-line description (max 200 chars)
- `keywords`: Array of searchable keywords (lowercase, specific)
- `applies_to`: Scope - `global`, `file:<path>`, or `area:<name>`
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

Memories should be atomic (one concept per file) with specific, actionable content. Scope should be carefully selected based on applicability.
