# Transcript Message Types

When processing Claude Code transcripts:

## Message Prefixes

- `[LOCAL_RECALL_INTERNAL]` - System instruction or analysis directive about the transcript format itself
  - These are NOT user prompts or assistant responses
  - Should be skipped during memory extraction
  - Appear at the beginning of condensed transcripts
  - Used to guide memory extraction logic

- `[User]` - Actual user request or prompt
- `[Assistant]` - Claude's response or analysis
- `[Tool: Name]` - Tool invocation (Read, Edit, Write, Bash, etc.)
- `[Result: OK/ERROR]` - Tool execution outcome

## Implications

When parsing transcripts for memory extraction:
1. Skip `[LOCAL_RECALL_INTERNAL]` blocks entirely
2. They won't contribute useful project memories
3. Focus on actual `[User]`, `[Assistant]`, and `[Tool]` entries
4. This prevents meta-analysis from polluting the memory store
