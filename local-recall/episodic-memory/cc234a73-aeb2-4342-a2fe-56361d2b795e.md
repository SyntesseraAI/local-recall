# Memory File Format and Deduplication

Each memory file contains YAML frontmatter with:
- `id`: Unique UUID identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable terms
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for sorting and deduplication
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

The `occurred_at` and `content_hash` fields are used together to detect and prevent duplicate memories. If a memory with the same timestamp and content hash already exists, it won't be created again.
