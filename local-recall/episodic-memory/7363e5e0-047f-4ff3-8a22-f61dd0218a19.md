The configuration system in `src/utils/config.ts` exports both `loadConfig` and `getConfig` functions. Multiple hook files import both functions, suggesting:
- `loadConfig` - Initializes/loads configuration from environment or `.local-recall.json`
- `getConfig` - Retrieves the already-loaded configuration object

This pattern appears across hooks: `src/hooks/user-prompt-submit.ts`, `src/hooks/user-prompt-submit-thinking.ts`, `src/hooks/session-start.ts`, and MCP server `src/mcp-server/server.ts`.

Both the episodic and thinking enabled flags integrate with this config loading pattern to control feature activation.
