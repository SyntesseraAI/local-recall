Memory files are stored in `local-recall/episodic-memory/` as markdown with YAML frontmatter. Required fields:
- `id`: UUID
- `subject`: Brief description
- `keywords`: Array of searchable terms
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

Memory files are version-controlled in git. The SQLite database (memory.sqlite) and recall.log are gitignored.
