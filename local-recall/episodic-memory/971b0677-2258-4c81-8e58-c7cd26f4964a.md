Each memory is a markdown file with structured YAML frontmatter containing metadata:
- `id`: UUID identifier
- `subject`: Brief description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

Memory files are version-controlled in git, stored in `local-recall/episodic-memory/*.md`
