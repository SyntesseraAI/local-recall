Memory files in Local Recall are stored as markdown files with YAML frontmatter in the `local-recall/episodic-memory/` directory and are tracked in git. Each file has metadata fields: `id` (UUID), `subject`, `keywords`, `applies_to` (scope), `occurred_at` (timestamp), and `content_hash` (SHA-256 prefix). The actual memory content follows the frontmatter. This format allows version control of project knowledge and enables sharing memories across team members.
