Local Recall stores memories in `local-recall/episodic-memory/` directory as markdown files with YAML frontmatter. Key characteristics:

- Each memory file contains metadata fields: `id`, `subject`, `keywords`, `applies_to`, `occurred_at`, `content_hash`
- Memory files ARE version-controlled and tracked in git
- The index file (`local-recall/index.json`) is gitignored and auto-generated
- `applies_to` field determines scope: `global`, `file:<path>`, or `area:<component>`
- `occurred_at` uses ISO-8601 timestamps for deduplication and sorting
- `content_hash` (SHA-256 prefix) prevents duplicate memory creation

This design allows memories to be shared among team members while keeping generated artifacts out of version control.
