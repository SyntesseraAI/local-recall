When extracting memories from Claude Code transcripts:

**What to extract:**
- Architectural decisions and reasoning
- Bug fixes and root cause analysis
- Code patterns or conventions specific to the project
- Configuration quirks or gotchas
- User preferences discovered
- Important component relationships
- Performance considerations

**What NOT to extract:**
- Generic programming knowledge
- Temporary debugging steps
- Obvious or trivial information
- Sensitive data
- Information that becomes outdated quickly

**Scope types:**
- `global` - Applies to entire codebase
- `file:<path>` - Specific to a file
- `area:<name>` - Related to a component

**Memory format:**
- subject: Brief one-line description (max 200 chars)
- keywords: 1-10 searchable keywords (lowercase, specific)
- applies_to: Scope string
- content: Detailed markdown content

Be specific with file paths and function names. Focus on actionable insights that help future assistants avoid mistakes or work more efficiently.
