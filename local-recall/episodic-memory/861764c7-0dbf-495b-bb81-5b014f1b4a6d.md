The project transitioned from using better-sqlite3 (native dependency with bundling issues) to Orama (pure JavaScript vector store). This eliminates process isolation issues and allows hooks to run directly without daemon processes. The UserPromptSubmit hook now handles both episodic and thinking memory search using Orama + Ollama embeddings, configured via environment variables (episodicEnabled, thinkingEnabled, etc.).
