Local Recall enables Claude Code and other AI tools to retain information and context across sessions through structured markdown files stored in `local-recall/episodic-memory/`. The system includes:

- **Architecture**: Core memory management (CRUD ops, vector store with SQLite, semantic search), MCP server for memory tools, Claude Code hooks (SessionStart, UserPromptSubmit, Stop)
- **Memory Format**: Markdown files with YAML frontmatter containing id, subject, keywords, applies_to (scope), occurred_at, content_hash
- **Storage**: Memories tracked in git at `local-recall/episodic-memory/*.md`, SQLite database at `local-recall/memory.sqlite` (gitignored)
- **Search**: Uses fastembed with BGE-small-en-v1.5 model (~133MB) for semantic similarity search with cosine distance scoring (0.0-1.0)
- **Hooks**: SessionStart loads 5 most recent memories; UserPromptSubmit performs semantic search on user input; Stop hook disabled (async processing via MCP daemon every 5 minutes)
- **MCP Tools**: memory_create, memory_get, memory_search, memory_list, index_rebuild
