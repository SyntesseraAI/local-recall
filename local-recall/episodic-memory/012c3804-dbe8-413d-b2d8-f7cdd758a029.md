Three hooks are configured in hooks.json:
1. **SessionStart**: Loads 5 most recent memories at session start
2. **UserPromptSubmit**: Performs semantic search when user submits prompt, initializes vector store lazily
3. **Stop**: Disabled - memory extraction now handled by MCP daemon

The MCP server exposes tools (memory_create, memory_get, memory_search, memory_list, index_rebuild) and runs a background daemon processing transcripts every 5 minutes.
