# Transcript Parsing Considerations

When parsing Claude Code transcripts, tool invocations and results can contain nested JSON/structured data that must be properly handled:

- Tool results may contain JSON objects or arrays
- Nested structures should be preserved in the transcript parsing logic
- The transcript format includes tool names, parameters, and results
- Proper parsing requires careful handling of string boundaries and nested structures

This is relevant when `transcript-collector.ts` processes JSONL transcript files to extract memories.
