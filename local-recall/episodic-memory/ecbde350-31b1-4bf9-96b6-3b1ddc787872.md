# Git Branch Naming Convention

The Local Recall project uses a specific git branch naming convention for refactoring work:

- Format: `claude/refactor-[description]-[UUID]`
- Example: `claude/refactor-memory-collection-01HQY8jXfJm7zh35BcYYiE1u`
- Current branch: `claude/refactor-memory-collection-01HQY8jXfJm7zh35BcYYiE1u`

## Purpose

This naming convention likely helps:
- Track Claude-assisted refactoring work
- Maintain unique branch identifiers
- Organize multiple concurrent refactoring efforts
- Make it easy to identify AI-assisted changes in git history

## Usage

When creating new branches for refactoring or feature work in this project, follow this pattern with:
1. `claude/` prefix to indicate AI assistance
2. Clear kebab-case description of the work
3. A unique identifier (UUID) to prevent naming collisions
