Memory files use YAML frontmatter with these required fields:
- `id`: UUID for unique identification
- `subject`: One-line description (max 200 chars)
- `keywords`: Array of searchable keywords (lowercase, 1-10 items)
- `applies_to`: Scope (`global`, `file:<path>`, or `area:<name>`)
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 prefix (16 chars) for deduplication

Memory content is markdown-formatted and stored in `.md` files under `local-recall/episodic-memory/`
