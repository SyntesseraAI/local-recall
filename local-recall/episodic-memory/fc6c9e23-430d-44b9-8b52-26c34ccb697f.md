The transcript-collector module must parse JSONL files (where each line is a separate JSON object). Key considerations:
- Handle file not found errors gracefully
- Parse line-by-line to avoid loading entire files into memory
- Each line may be a partial or complete transcript event
- File may not exist initially or may contain partial data
- Error handling should allow graceful degradation if transcripts cannot be read
