Memory files use YAML frontmatter with specific required fields:

**Required Fields:**
- `id`: UUID for unique identification
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords for discovery
- `applies_to`: Scope indicator - `global`, `file:<path>`, or `area:<name>`
- `occurred_at`: ISO-8601 timestamp for deduplication and sorting
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

**Important Notes:**
- Memory files ARE version-controlled - they will be committed to git
- Memories should be atomic - one concept per file
- Use specific, lowercase keywords for better searchability
- The index.json file is gitignored and auto-generated
- Deduplication uses both `occurred_at` and `content_hash` to detect duplicates
