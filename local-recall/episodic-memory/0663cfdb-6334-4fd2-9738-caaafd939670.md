# Transcript File Format

Claude's transcript files are stored in JSONL (JSON Lines) format, where each line is a separate JSON object.

## Message Structure

Each message in a transcript has:
- `role`: Either "user" or "assistant"
- `content`: The message content (varies by type)
- Additional metadata fields depending on role

## Content Types

- Text messages: Simple string content
- Tool invocations: Structured content with tool name and parameters
- Tool results: Success/error outcomes with result data

## Parsing Considerations

When processing transcripts:
1. Read file line-by-line (each line is a complete JSON object)
2. Parse each JSON object independently
3. Handle both text and structured content appropriately
4. Process in order to maintain conversation flow
