Memory management and version control strategy:

- **Memory files** (`local-recall/episodic-memory/*.md`): ARE version-controlled and committed to git
- **SQLite database** (`local-recall/memory.sqlite`): gitignored and auto-generated from memory files
- **Vector embeddings**: Generated automatically when memories added to store, not stored in git
- **Recall log** (`local-recall/recall.log`): gitignored debug log file
- **Gitignore file** (`local-recall/.gitignore`): Auto-generated by utils/gitignore.ts, excludes memory.sqlite and recall.log

This approach makes memories shareable among team members while keeping generated data (database, embeddings) local.
