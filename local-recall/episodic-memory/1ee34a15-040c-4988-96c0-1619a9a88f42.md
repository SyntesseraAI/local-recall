# Transcript Structure and Format

Claude's cached transcripts contain:

1. **Session Boundaries**: Clear markers indicating when sessions start and end
2. **Tool Invocations**: Detailed records of all tools used (Read, Edit, Write, Bash, Grep, etc.)
3. **Results**: Outcomes of tool calls (OK, ERROR, output data)
4. **User/Assistant Messages**: Structured dialogue with clear roles
5. **Metadata**: Project context, working directory, timestamps

This structure is important for:
- Accurate memory extraction
- Understanding what was accomplished in a session
- Tracking tool usage patterns and patterns of code changes
- Reconstructing the flow of a session for context

The transcript-collector needs to handle this structured format and parse it correctly for the transcript-condenser and memory-extractor.
