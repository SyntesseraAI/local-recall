MCP search tools (`episodic_search`, `thinking_search`) now return full memory content in results. Added `max_tokens` parameter to tool schemas (default 2000) to limit total tokens in response. Results include: `id`, `subject`, `content`, `occurred_at`, `similarity_score`, and `tokens_used`. This enables clients to enforce token budgets while accessing full memory content.
