The MCP server (`src/mcp-server/server.ts`) exposes memory management tools and runs an autonomous background daemon:

**Available Tools**:
- `memory_create`: Create new memory (idempotent)
- `memory_get`: Retrieve specific memory by ID
- `memory_search`: Semantic search using embeddings
- `memory_list`: List memories with optional filtering
- `index_rebuild`: Sync vector store with files

**Background Daemon**:
- Syncs transcripts from Claude's cache (`~/.claude/projects/<project>/transcripts/`)
- Extracts memories from transcripts using `claude -p`
- Tracks processed transcripts with content hashes
- Deletes and recreates memories when transcripts change
- Runs every 5 minutes autonomously
- Memory extraction moved from Stop hook to daemon for better reliability
