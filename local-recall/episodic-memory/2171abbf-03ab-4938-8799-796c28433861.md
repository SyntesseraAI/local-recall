# Transcript Collector Implementation

The `transcript-collector.ts` module is responsible for:

1. **Syncing transcripts** from `~/.claude/projects/{folder}/transcripts/` to the local memory system
2. **Processing JSONL files** that contain Claude Code session transcripts
3. **Running asynchronously** as part of the MCP server daemon (every 5 minutes)
4. **Extracting memories** from transcripts using Claude's analysis
5. **Tracking processed transcripts** with content hashes to detect changes

- Read JSONL transcript files from Claude's cache
- Use `claude -p` CLI or memory_create tool to extract and store memories
- Handle deduplication via content hashes
- Delete and recreate memories when transcripts change
