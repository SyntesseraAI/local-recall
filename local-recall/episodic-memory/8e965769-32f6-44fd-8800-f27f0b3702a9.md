Each memory markdown file requires YAML frontmatter with these fields:

- `id`: UUID unique identifier
- `subject`: Brief one-line description
- `keywords`: Array of searchable keywords (lowercase)
- `applies_to`: Scope - `global`, `file:<path>`, or `area:<name>`
- `occurred_at`: ISO-8601 timestamp (used for deduplication and sorting)
- `content_hash`: SHA-256 hash prefix (16 chars) for deduplication

Memory files are atomic - each should focus on one concept. Keywords should be specific for better searchability.
