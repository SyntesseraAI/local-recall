Episodic and thinking memories are completely independent systems with separate vector stores, search implementations, and configuration.

**Episodic Memories**:
- Core memory storage in `src/core/episodic-jsonl-store.ts`
- Vector store: `src/core/vector-store.ts` (Orama-based)
- Search: `src/core/search.ts`
- Configuration: `episodicEnabled`, `episodicMaxTokens`, `episodicMinSimilarity`

**Thinking Memories**:
- Separate storage in `src/core/thinking-jsonl-store.ts`
- Vector store: `src/core/thinking-vector-store.ts` (distinct Orama index)
- Search: `src/core/thinking-search.ts`
- Configuration: `thinkingEnabled`, `thinkingMaxTokens`, `thinkingMinSimilarity`

**Index Files**:
- Episodic: `local-recall/orama-episodic-index.json`
- Thinking: `local-recall/orama-thinking-index.json`
- Both are gitignored and auto-generated

**UserPromptSubmit Hook**:
- Unified hook that searches both types based on configuration
- Each memory type has independent thresholds and token budgets
- Results are combined and injected into context
- Allows enabling/disabling each type independently
