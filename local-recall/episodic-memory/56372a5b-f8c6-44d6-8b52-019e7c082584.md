# Transcript Condensing

Transcript condensing is a preprocessing step that converts verbose JSONL transcripts into a more concise event-based format.

## Purpose

- Reduces token usage when passing transcripts to Claude for memory extraction
- Maintains essential information while removing redundant fields
- Makes transcripts easier to analyze for pattern extraction

## Format

Condensed format uses event markers:
- `[User]` - User messages
- `[Assistant]` - Assistant responses
- `[Tool: Name]` - Tool invocations with name
- `[Result: OK/ERROR]` - Tool execution outcomes

## Implementation

The condensing logic is implemented in `src/core/transcript-collector.ts` and converts JSONL format to this event-based representation.
