# Memory File Schema

Each memory file uses YAML frontmatter with the following fields:

## Required Fields
- `id`: Unique identifier (UUID)
- `subject`: Brief one-line description of the memory
- `keywords`: Array of searchable keywords (1-10 items, lowercase, specific)
- `applies_to`: Scope - `global`, `file:<path>`, or `area:<name>`
- `occurred_at`: ISO-8601 timestamp of when the event occurred
- `content_hash`: SHA-256 hash prefix (16 chars) of the content for deduplication

## Memory File Location
- All memory files are stored in `local-recall/episodic-memory/` directory
- Files are markdown format with YAML frontmatter
- Memory files ARE version-controlled and committed to git
- The `local-recall/index.json` is gitignored and auto-generated
