Each memory file is a markdown document with YAML frontmatter containing:
- `id`: UUID
- `subject`: Brief description
- `keywords`: Array of searchable keywords
- `applies_to`: Scope (global, file:<path>, or area:<name>)
- `occurred_at`: ISO-8601 timestamp
- `content_hash`: SHA-256 prefix (16 chars) for deduplication

Memory files are atomic (one concept per file) and ARE version-controlled in git.
