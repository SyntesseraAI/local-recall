Local Recall uses JSONL (JSON Lines) format for storing memories with multi-file support:

- Episodic memories stored in `local-recall/episodic-000001.jsonl`, `episodic-000002.jsonl`, etc.
- Thinking memories stored in `local-recall/thinking-000001.jsonl`, `thinking-000002.jsonl`, etc.
- Files use 6-digit zero-padded numbering for proper sorting
- Each line is a complete JSON object representing one memory
- Allows better scalability than individual markdown files
- Migration involved converting from individual markdown files to JSONL format
