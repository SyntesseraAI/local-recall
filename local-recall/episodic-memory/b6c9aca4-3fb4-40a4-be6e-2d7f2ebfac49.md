The memory extraction process emphasizes creating specific (with file paths and concrete details), actionable (helping future assistants avoid mistakes), and concise (one concept per memory) memories. Generic programming knowledge, temporary debugging steps, and trivial information are explicitly excluded to maintain memory quality.
