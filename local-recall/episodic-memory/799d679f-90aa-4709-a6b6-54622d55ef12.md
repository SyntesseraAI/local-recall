# Transcript JSONL Structure

Claude's transcript JSONL files contain metadata at both the file and individual turn levels.

## File-Level Metadata

Each JSONL file starts with metadata lines that include:
- `created_at` - When the transcript was created
- `model_identifier` - The Claude model used (e.g., "claude-opus-4-1")
- `transcript_id` - Unique identifier for the transcript
- `turns` - Array containing conversation turns

## Turn Structure

Each turn in the `turns` array contains:
- `turn_type` - Either "user" or "assistant"
- `turn` - The turn number (incremental)
- `content` - The actual message content
- `metadata` - Turn-specific metadata
- `raw_content` - Raw message data

## Parsing Considerations

- Each line is a complete JSON object (JSONL format)
- Must parse each line independently
- Metadata is intermixed with turn data
- The structure follows Claude's internal transcript format
