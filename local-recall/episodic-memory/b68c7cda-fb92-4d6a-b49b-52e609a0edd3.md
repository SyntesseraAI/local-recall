The MCP server provides memory tools and maintains a background daemon:

1. **Available Tools**:
   - `memory_create`: Create new memory (idempotent)
   - `memory_get`: Retrieve memory by ID
   - `memory_search`: Semantic search using vector embeddings
   - `memory_list`: List memories with optional filtering
   - `index_rebuild`: Sync vector store with memory files

2. **Background Daemon**:
   - Syncs transcripts from Claude's cache (`~/.claude/projects/<project>/transcripts/`)
   - Processes transcripts using `claude -p` to extract memories
   - Tracks processed transcripts with content hashes
   - Deletes and recreates memories when transcripts change
   - Runs every 5 minutes

3. **Configuration**:
   - Can be configured via environment variables or `.local-recall.json`
   - `LOCAL_RECALL_DIR` environment variable specifies memory directory
