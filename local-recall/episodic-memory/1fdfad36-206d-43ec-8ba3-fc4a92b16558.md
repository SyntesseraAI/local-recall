The MCP server includes a background daemon with these behaviors:

- **Polling**: Runs every 5 minutes to process transcripts
- **Source**: Syncs transcripts from Claude's cache (`~/.claude/projects/<project>/transcripts/`)
- **Processing**: Uses `claude -p` command to extract memories from transcripts
- **Deduplication**: Tracks processed transcripts using content hashes for change detection
- **Updates**: Deletes and recreates memories when transcripts change

This asynchronous approach avoids blocking the Stop hook and allows for more sophisticated memory extraction using the Claude CLI's processing capabilities.
