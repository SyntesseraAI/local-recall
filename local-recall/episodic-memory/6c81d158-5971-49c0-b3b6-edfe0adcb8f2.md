Added two environment variables to control memory systems:

- `LOCAL_RECALL_EPISODIC_ENABLED` (default: `false`) - Controls episodic memory retrieval in hooks and extraction in MCP server
- `LOCAL_RECALL_THINKING_ENABLED` (default: `true`) - Controls thinking memory retrieval in hooks and extraction

These flags are parsed in `src/utils/config.ts` and added to the config schema in `src/core/types.ts`. The flags are used in:
- `src/hooks/user-prompt-submit.ts` - Skips episodic memory injection if disabled
- `src/hooks/user-prompt-submit-thinking.ts` - Skips thinking memory injection if disabled
- `src/hooks/session-start.ts` - Skips loading episodic memories if disabled
- `src/mcp-server/server.ts` - Skips extraction and processing of disabled memory types

Design decision: Episodic defaults to OFF (optional feature), thinking defaults to ON (core feature).
